# 质量评估与规则优先级

> 本文件包含量化评估标准、规则冲突优先级和质量检查清单。由 SKILL.md 按需加载。

说明：自然表达不单独作为第 6 个评分维度，而是并入可读性与教学性。任何“去 AI 味”改写都不得以删减事实、边界说明、学习元素或可执行细节为代价。

---

## 规则冲突优先级

当多条规则同时适用于同一段文本时，按以下优先级处理（高优先级覆盖低优先级）：

| 优先级 | 规则 | 说明 |
| -------- | ------ | ------ |
| **P0** | 代码块内不检查 | ` ``` ` 或 `~~~` 包裹的区域完全跳过，包括空格、标点、术语 |
| **P1** | 行内代码内不检查 | `` ` `` 包裹的内容不检查中英空格，如 `apiKey` |
| **P2** | LaTeX 公式内不检查 | `$...$` 和 `$$...$$` 视为代码，不检查不翻译 |
| **P3** | YAML frontmatter 不检查 | `---` 包裹的元数据区域跳过 |
| **P4** | URL/路径保持原样 | `https://api.example.com` 内不加空格 |
| **P5** | 术语表 > 通用规则 | 术语表标注 `keep_en: true` 的术语保留英文 |
| **P6** | 中英文空格规则 | 正文中中英文 / 中文数字之间必须加空格 |
| **P7** | 术语首次标注规则 | 每个术语首次出现时标注中文翻译 |
| **P8** | Markdown 换行保留 | 行尾恰好 2 个空格视为 `<br>`，不视为多余空格 |

### 实际冲突示例

| 场景 | 适用规则 | 结果 |
| ------ | ---------- | ------ |
| `const apiKey = "sk-..."` 在代码块内 | P0 | 不检查，保持原样 |
| 正文中写 `apiKey` | P1 | 行内代码内不加空格 |
| `使用 Docker 部署` | P6 | 中英之间有空格 ✓ |
| 首次出现 API | P7 | 写作 `API（应用程序接口）` |
| 术语表中 `Docker` 在 no_translate 列表 | P5 | 不翻译，不标注中文 |

## 量化评估标准

每篇文档按以下 5 个维度评分，总分 100 分：

| 维度 | 权重 | 评分标准 |
| ------ | ------ | ---------- |
| **结构性** | 20% | 标题层级正确、目录清晰、逻辑连贯、导航完整 |
| **准确性** | 25% | 技术内容正确、术语使用一致、代码可运行、链接有效 |
| **可读性** | 25% | 中英文混排规范、段落适中、排版舒适、自然表达、格式统一 |
| **教学性** | 20% | 有学习目标、解释"为什么"、包含练习、递进合理 |
| **实用性** | 10% | 示例贴近真实、常见问题覆盖、错误处理清晰 |

### 评分细则

```text
结构性（20分）：
  - 标题层级无跳跃（5分）
  - 有清晰的章节划分（5分）
  - 内容逻辑递进合理（5分）
  - 有目录/导航/前后文链接（5分）

准确性（25分）：
  - 技术描述无事实错误（10分）
  - 术语使用一致（5分）
  - 代码示例完整可运行（5分）
  - 链接有效、版本信息准确（5分）

可读性（25分）：
  - 中英文之间有空格（5分）
  - 正文用中文标点、代码保留英文标点（5分）
  - 段落与句长适中，不过度堆叠短句（5分）
  - 叙述自然，无明显模板腔、生成式转场、自我声明堆积（5分）
  - 列表/表格/代码块使用合理，不过度为凑结构而堆砌（5分）

教学性（20分）：
  - 有明确的学习目标（5分）
  - 核心概念解释了"为什么"（5分）
  - 包含练习或实践环节（5分）
  - 难度递进合理（5分）

实用性（10分）：
  - 示例来自真实场景（4分）
  - 包含常见问题解答（3分）
  - 有错误处理和排查指引（3分）
```

### 可读性量化指标（辅助参考）

- 平均句长：15-35 字/句为宜（>50 字应拆分）
- 段落密度：每段 3-8 行正文（不含代码块）
- 信息密度：每段引入不超过 2 个新概念
- 标题间距：两个同级标题间建议 50-300 字正文
- 代码文本比：教程类建议 30%-50%，参考类可达 60%

### 自然度辅助检查（并入可读性/教学性）

- 元话语密度：连续两段内不应反复出现“换句话说”“更准确地说”“这里需要注意”“最重要的是”等生成式转场。
- 教学控制句密度：避免每一小节都用“如果你只想”“读到这里你应该”打断正文。
- 列表自然度：多条列表不要句式完全对称；若信息密度过低，应合并或改写为正文。
- 结构块自然度：学习目标、自测、注意事项可以保留，但导语要像正文自然引出，而不是表单标签堆叠。
- 稳分要求：若做了“去 AI 味”润色，复评分不得低于处理前。

## 去 AI 味门槛（不新增维度）

自然表达不单独新增第六维，而是作为 **可读性 + 教学性** 的强制门槛处理。评分时同时看词面、句式和节奏，不只看有没有“本文将”。

要拿到 A / S，尤其是满分 100，必须同时满足下面 8 条：

1. 无明显生成式转场句堆叠，例如“更准确地说 / 换句话说 / 最重要的是 / 如果你……”连续出现。
2. 无抽象套话密集堆叠，例如“核心价值 / 落地边界 / 能力闭环 / 体系化 / 赋能 / 可发现、可复用、可验证”成组出现。
3. 无高频二元口号句，例如“不是 A，而是 B”“不只是 A，更是 B”“真正 X 的不是 Y，而是 Z”反复推进。
4. 无机械总结句连续收束段落，例如“这意味着 / 这解释了 / 这种路径的价值 / 这套顺序的本质”。
5. 无过强作者在场感；非评论文中避免反复出现“我认为 / 我更愿意 / 我会 / 我倾向于”。分析型文章可以少量保留操作性第一人称，用来表达取舍，而不是展示作者存在。
6. 学习目标、阅读指引、自测问题等结构块服务内容，而不是为了凑模板存在。
7. 列表和小标题不过度整齐，不出现低信息密度的对称句式堆叠。
8. 去 AI 味后五维总分不下降，且没有删除事实边界、版本差异、错误条件或 caveat。

若上述门槛任一项未满足：

- `可读性` 上限封顶为 20/25。
- 总分不得超过 89 分。
- 不得评为 S 级。

辅助判断：若一段话删掉“价值、能力、体系、边界、赋能、可复用”等抽象词后几乎没有具体信息，说明它仍有明显 AI 味，应改写为对象、动作或场景判断。

## 分析型技术文章附加检查

适用于开源项目解读、架构分析、benchmark 解读、系统评测等文体；不适用于普通 API 文档或操作手册。

满足越多，越容易达到高完成度的专家级成稿：

- 开头先给判断，而不是先给背景材料、Stars、版本号或功能列表。
- 正文前 20% 给出总览地图：图、表或明确总览段至少一种。
- 当系统包含两套以上并行机制时，先拆边界，再展开机制细节。
- 至少有一个“任务如何流过系统”的具体案例，而不是全篇静态结构说明。
- benchmark 段解释“测的是什么、对应哪部分系统、不能推出什么”。
- 结尾给采用顺序、适用边界或决策建议。

若是分析型文章且以上 6 项中低于 3 项满足：

- `结构性` 上限封顶为 16/20。
- `实用性` 上限封顶为 7/10。
- 不建议评为 S 级。

### 评分等级

| 等级 | 分数范围 | 含义 |
| ------ | ---------- | ------ |
| S | 90-100 | 专家级，可作为范例 |
| A | 80-89 | 优秀，满足专业标准 |
| B | 70-79 | 良好，有改进空间 |
| C | 60-69 | 及格，需要优化 |
| D | <60 | 不合格，需要重写 |

### 发布门槛

| 等级 | 发布建议 |
| ------ | ---------- |
| S/A（≥80） | ✅ 可直接发布 |
| B（70-79） | ⚠️ 可发布，但建议先修复主要扣分项 |
| C（60-69） | ❌ 不建议发布，需执行 `optimize-cn-doc` 流程优化 |
| D（<60） | ❌ 禁止发布，需重写或使用 `write-cn-doc` 重新生成 |

> **强制规则**：当 AI 生成的文档评分低于 B 级（70 分）时，必须在输出中明确标注警告：
> `⚠️ 本文档评分 XX 分（C/D 级），建议优化后再发布。`

## 质量检查清单

```markdown
### 结构
- [ ] 标题层级正确（# > ## > ### > ####，无跳跃）
- [ ] 目录/导航完整
- [ ] 代码块语法正确且指定了语言
- [ ] 所有链接有效

### 内容
- [ ] 术语使用一致
- [ ] 每个核心概念都解释了"为什么"
- [ ] 代码示例完整可运行
- [ ] 包含常见问题解答

### 格式
- [ ] 中英文之间有空格
- [ ] 数字与中文之间有空格
- [ ] 中文语境用中文标点
- [ ] 列表格式统一（有序/无序不混用）

### 自然度
- [ ] 不存在连续堆叠的元话语或模板腔转场
- [ ] 无抽象套话密集堆叠、二元口号句、高光总结句
- [ ] 列表、标题和导语不是机械对称的占位结构
- [ ] 未因“去 AI 味”删除边界说明、验收条件、事实差异或学习元素

### 教学性（标准/完整模式）
- [ ] 有明确的学习目标
- [ ] 包含练习或实践环节
- [ ] 难度递进合理
- [ ] 有进阶路径指引
```
