# 质量评估与规则优先级

> 本文件包含量化评估标准、规则冲突优先级和质量检查清单。由 SKILL.md 按需加载。

---

## 规则冲突优先级

当多条规则同时适用于同一段文本时，按以下优先级处理（高优先级覆盖低优先级）：

| 优先级 | 规则 | 说明 |
|--------|------|------|
| **P0** | 代码块内不检查 | ` ``` ` 或 `~~~` 包裹的区域完全跳过，包括空格、标点、术语 |
| **P1** | 行内代码内不检查 | `` ` `` 包裹的内容不检查中英空格，如 `apiKey` |
| **P2** | LaTeX 公式内不检查 | `$...$` 和 `$$...$$` 视为代码，不检查不翻译 |
| **P3** | YAML frontmatter 不检查 | `---` 包裹的元数据区域跳过 |
| **P4** | URL/路径保持原样 | `https://api.example.com` 内不加空格 |
| **P5** | 术语表 > 通用规则 | 术语表标注 `keep_en: true` 的术语保留英文 |
| **P6** | 中英文空格规则 | 正文中中英文 / 中文数字之间必须加空格 |
| **P7** | 术语首次标注规则 | 每个术语首次出现时标注中文翻译 |
| **P8** | Markdown 换行保留 | 行尾恰好 2 个空格视为 `<br>`，不视为多余空格 |

### 实际冲突示例

| 场景 | 适用规则 | 结果 |
|------|----------|------|
| `const apiKey = "sk-..."` 在代码块内 | P0 | 不检查，保持原样 |
| 正文中写 `apiKey` | P1 | 行内代码内不加空格 |
| `使用 Docker 部署` | P6 | 中英之间有空格 ✓ |
| 首次出现 API | P7 | 写作 `API（应用程序接口）` |
| 术语表中 `Docker` 在 no_translate 列表 | P5 | 不翻译，不标注中文 |

## 量化评估标准

每篇文档按以下 5 个维度评分，总分 100 分：

| 维度 | 权重 | 评分标准 |
|------|------|----------|
| **结构性** | 20% | 标题层级正确、目录清晰、逻辑连贯、导航完整 |
| **准确性** | 25% | 技术内容正确、术语使用一致、代码可运行、链接有效 |
| **可读性** | 25% | 中英文混排规范、段落适中、排版舒适、格式统一 |
| **教学性** | 20% | 有学习目标、解释"为什么"、包含练习、递进合理 |
| **实用性** | 10% | 示例贴近真实、常见问题覆盖、错误处理清晰 |

### 评分细则

```text
结构性（20分）：
  - 标题层级无跳跃（5分）
  - 有清晰的章节划分（5分）
  - 内容逻辑递进合理（5分）
  - 有目录/导航/前后文链接（5分）

准确性（25分）：
  - 技术描述无事实错误（10分）
  - 术语使用一致（5分）
  - 代码示例完整可运行（5分）
  - 链接有效、版本信息准确（5分）

可读性（25分）：
  - 中英文之间有空格（5分）
  - 正文用中文标点、代码保留英文标点（5分）
  - 段落长度适中（3-8行）（5分）
  - 列表/表格/代码块使用合理（5分）
  - 关键概念有加粗/高亮标记（5分）

教学性（20分）：
  - 有明确的学习目标（5分）
  - 核心概念解释了"为什么"（5分）
  - 包含练习或实践环节（5分）
  - 难度递进合理（5分）

实用性（10分）：
  - 示例来自真实场景（4分）
  - 包含常见问题解答（3分）
  - 有错误处理和排查指引（3分）
```

### 可读性量化指标（辅助参考）

- 平均句长：15-35 字/句为宜（>50 字应拆分）
- 段落密度：每段 3-8 行正文（不含代码块）
- 信息密度：每段引入不超过 2 个新概念
- 标题间距：两个同级标题间建议 50-300 字正文
- 代码文本比：教程类建议 30%-50%，参考类可达 60%

### 评分等级

| 等级 | 分数范围 | 含义 |
|------|----------|------|
| S | 90-100 | 专家级，可作为范例 |
| A | 80-89 | 优秀，满足专业标准 |
| B | 70-79 | 良好，有改进空间 |
| C | 60-69 | 及格，需要优化 |
| D | <60 | 不合格，需要重写 |

### 发布门槛

| 等级 | 发布建议 |
|------|----------|
| S/A（≥80） | ✅ 可直接发布 |
| B（70-79） | ⚠️ 可发布，但建议先修复主要扣分项 |
| C（60-69） | ❌ 不建议发布，需执行 `optimize-cn-doc` 流程优化 |
| D（<60） | ❌ 禁止发布，需重写或使用 `write-cn-doc` 重新生成 |

> **强制规则**：当 AI 生成的文档评分低于 B 级（70 分）时，必须在输出中明确标注警告：
> `⚠️ 本文档评分 XX 分（C/D 级），建议优化后再发布。`

## 质量检查清单

```markdown
### 结构
- [ ] 标题层级正确（# > ## > ### > ####，无跳跃）
- [ ] 目录/导航完整
- [ ] 代码块语法正确且指定了语言
- [ ] 所有链接有效

### 内容
- [ ] 术语使用一致
- [ ] 每个核心概念都解释了"为什么"
- [ ] 代码示例完整可运行
- [ ] 包含常见问题解答

### 格式
- [ ] 中英文之间有空格
- [ ] 数字与中文之间有空格
- [ ] 中文语境用中文标点
- [ ] 列表格式统一（有序/无序不混用）

### 教学性（标准/完整模式）
- [ ] 有明确的学习目标
- [ ] 包含练习或实践环节
- [ ] 难度递进合理
- [ ] 有进阶路径指引
```
