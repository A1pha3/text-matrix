# 命令详细执行流程

> 本文件包含四个命令的完整执行步骤。由 SKILL.md 按需加载。

---

## `write-cn-doc` 执行步骤

```text
1. 场景路由 → 确定模式（快速/标准/完整）
2. 判断文体：教程 / 参考 / 分析
3. 信息收集：
   - 文档主题和技术栈
   - 目标读者（谁？当前水平？）
   - 文档目的（说明？教学？参考？）
4. [标准/完整模式] 确定参数：
   - 学习路径：用户路径 / 开发路径 / 进阶路径
   - 文档级别：Level 1⭐ / Level 2⭐⭐ / Level 3⭐⭐⭐ / Level 4⭐⭐⭐⭐
5. [分析型] 加载 blog-deep-dive.md，提前确定：核心判断 / 总览图 / 任务流案例 / benchmark 解读 / 采用建议
6. 选择模板：按级别从 templates.md 选择对应模板
7. 编写文档：按模板结构填充内容
8. 质量检查：加载 quality.md 执行量化评估
9. [分析型] 成稿增强：补总览地图、机制边界、任务流案例、benchmark 解释和采用顺序
10. 去 AI 味回路：清理模板腔 / 机械转场 / 过强作者在场感，但不删信息边界
11. 复评分：确认五维分数不低于第 8 步
12. 输出：完整文档 + 质量评分
```

## `translate-cn` 执行步骤

```text
1. 分析原文：
   - 统计总行数、代码块数、链接数
   - 识别保留区域（代码块、URL、命令、文件路径）
   - 标注专业术语（对照 terminology.json）
2. 翻译执行：
   - 保留代码块和命令原文
   - 术语按术语表翻译（首次出现：中文 + 英文原文）
   - 段落按中文表达习惯重组，而非逐句直译
3. 教学性增强（可选）：
   - 补充背景解释
   - 添加中文示例
   - 增加"为什么"的解释
4. 格式校验：
   - 中英文空格检查
   - 标点符号检查
   - 术语一致性检查
5. 输出：翻译文档 + 术语使用报告
```

### 术语使用报告格式

```markdown
## 术语使用报告
| 英文原文 | 中文翻译 | 出现次数 | 一致性 |
|----------|----------|----------|--------|
| {term} | {translation} | N | ✅/❌ |
总计 X 个术语，一致率 Y%
```

## `optimize-cn-doc` 执行步骤

```text
1. 诊断现状：
   - 加载 quality.md 执行量化评估，得到基线分数
   - 识别最大扣分项（优先修复）
2. 判断文体：教程 / 参考 / 分析
3. [分析型] 加载 blog-deep-dive.md，识别是否缺少：总览图、任务流案例、benchmark 解读、采用顺序
4. 改进执行（按扣分权重排序）：
   - 结构优化（标题层级、目录清晰度）
   - 内容增强（添加"为什么"、补充示例）
   - 格式修复（术语一致、中英文空格、标点）
   - 教学元素（学习目标、练习、常见问题）
5. 去 AI 味回路：
   - 清理模板腔、生成式转场、第一人称介入、机械列表
   - 不允许因去味而牺牲结构、准确性或教学性
6. 再次评估：执行量化评估得到优化后分数
7. 输出：优化后文档 + 改进报告
```

### 改进报告格式

```markdown
## 📋 优化报告

| 维度 | 优化前 | 优化后 | 变化 |
|------|--------|--------|------|
| 结构性 | X/20 | Y/20 | +Z |
| 准确性 | X/25 | Y/25 | +Z |
| 可读性 | X/25 | Y/25 | +Z |
| 教学性 | X/20 | Y/20 | +Z |
| 实用性 | X/10 | Y/10 | +Z |
| **总分** | **X** | **Y** | **+Z** |

### 主要改进
1. [改进项]：[具体做了什么] → [效果]
2. ...

### 去 AI 味处理
- 状态：✅ 已完成 / ⚠️ 未完成（原因）
- 主要处理：[删除了哪些元话语 / 重写了哪些模板化导语 / 如何避免口语化过度]

### 遗留问题
- [如有未修复项及原因]
```

## `enhance-learning` 执行步骤

```text
1. 分析现有文档结构
2. 确定可增强的位置：
   - 章节开头 → 添加学习目标（基础/进阶/专家三级）
   - 概念讲解 → 添加"为什么"和类比解释
   - 代码示例后 → 添加练习（理解型/应用型/分析型）
   - 章节结尾 → 添加自测检查清单
   - 文档结尾 → 添加进阶路径指引
3. 生成增强内容
4. 输出：增强后文档 + 增强元素清单
```

### 增强清单格式

```markdown
## 📋 增强清单
| 位置 | 增强类型 | 内容摘要 |
|------|----------|----------|
| §{N} 开头 | 学习目标 | {目标描述} |
| §{N} 后 | 练习 | {练习类型和数量} |
| 文档结尾 | 自测检查 | {检查项数量} |
| 文档结尾 | 进阶路径 | {路径描述} |
```
