# YouTube 视频深度解读工作流

将 YouTube 视频转化为可发布的中文深度解读文章，核心原则是先取证，再写作。

## 0. 触发与默认动作

- 用户消息只要包含以下任一形式，就视为本 skill 的触发信号：
  - `https://www.youtube.com/watch?v=...`
  - `https://youtu.be/...`
  - `https://www.youtube.com/embed/...`
  - `https://www.youtube.com/shorts/...`
  - B站视频链接 `https://www.bilibili.com/video/...`
  - 纯视频ID（如 `dQw4w9WgXcQ`）
  - 视频标题（如 "AI Agents, Clearly Explained"）
- 如果消息里只有 YouTube/B站 链接，或只有链接加上"写文章""深度解读""整理成视频笔记"之类短指令，默认激活本 skill 并直接进入"写作"模式。
- 收到视频内页面链接时（如 playlist、channel），先归一化为视频播放页面，再继续取证与写作。
- 只有当用户明确要求"只分析不写""只给摘要""只做发布"时，才覆盖默认写作动作。
- 不要因为用户只给了链接就先追问题目、角度或受众；应先基于视频证据完成首版可读草稿，再在必要时补充边界说明。

## 1. 适用边界

### 适合

- 用户提供 YouTube/B站 视频链接、视频ID，或要求分析某个视频内容
- 目标产物是中文视频深度解读、视频笔记、学习指南
- 目标站点是 text-matrix 或兼容 Hugo Frontmatter 的内容库
- 视频内容为：技术教程、知识分享、课程讲座、产品评测、行业分析等

### 不适合

- 纯娱乐、vlog、新闻现场报道
- 没有视频证据、只有模糊想法的营销文案
- 要求伪造视频数据、观看量、评论内容或频道信息

## 2. 先做路由

收到任务后，先判断任务范围。

| 模式 | 触发信号 | 输出 |
|------|----------|------|
| 分析 | "看看这个视频值不值得写""帮我分析这个视频" | 视频信息卡 + 选题建议 |
| 写作 | "根据这个视频写文章""整理成视频笔记" | 完整文章草稿 |
| 发布 | 明确要求"发布到 text-matrix / 飞书 / 更新索引" | 文章文件 + 发布结果 |

默认执行"写作"模式；如果输入只有视频链接，也直接进入"写作"模式；只有明确要求时才进入"发布"模式。

## 3. 执行流程

### Step 1: 视频取证

先从 YouTube/B站 页面获取视频证据，再决定文章结构。

如果用户提供的是视频内子页面链接（如 playlist、channel），先提取并记录：

- 原始链接
- 归一化后的视频播放页面链接
- 对应的视频ID或视频标识

最低取证项：

- 视频标题与链接
- 频道名称与订阅数
- 视频发布时间与时长
- 观看量、点赞数、评论数（如果可获取）
- 视频描述中的关键信息（定位、能力、章节时间戳）
- 视频章节（如果有）
- 视频语言（根据标题和描述判断）

可选补充项：

- 热门评论内容（反应观众关注点）
- 相关视频推荐（了解同系列或同话题内容）
- 字幕/ transcript（如果可获取并有参考价值）
- 频道其他热门视频

取证原则：视频描述足够时不必深挖字幕；描述不足时再查看评论和章节；无法确认的信息标记为"未知"或"未在视频页面明确说明"，禁止补写。

### Step 2: 选题定位

根据证据判断文章主轴，只保留一个主叙事。

优先级如下：

1. 知识密度高、干货多：适合写"核心要点提炼 + 深度解读"
2. 实操性强、有步骤演示：适合写"教程型笔记 + 实战指南"
3. 观点新颖、有独特见解：适合写"观点解析 + 思考延伸"
4. 内容较新但信息有限：适合写"视频导读 + 观看建议"

选题输出至少包含：

- 目标读者
- 一句话标题方向
- 为什么值得写
- 本文不覆盖什么

### Step 3: 生成文章草稿

进入写作阶段时，必须加载 [../cn-doc-writer/SKILL.md](../cn-doc-writer/SKILL.md)，再将视频证据整理为写作输入。文章默认定位为：

- 分类：视频精读
- 文风：专业、克制、可验证
- 目标：帮助读者判断视频价值并提取关键信息

推荐结构：

1. 视频概览（标题、频道、时长、观看量、发布时间）
2. 学习目标或读者收益
3. 视频核心内容梳理（按时间线或主题）
4. 关键要点提炼（3-5个核心观点）
5. 重要时间戳/章节索引
6. 适用观众、优势与局限
7. 常见问题或观看建议
8. 总结与延伸阅读

结构不是硬性模板；若视频信息不足，优先缩短而不是补写空章节。

### Step 4: 三轮事实校验

草稿完成后，必须按顺序完成以下检查：

1. 事实一致性：视频标题、频道名、观看量、数据、命令、时间戳与视频页面一致
2. 术语一致性：同一概念的中英文叫法保持统一，首次出现时可保留原文并加中文括注
3. 证据边界：删除所有无来源的统计数据、观点引用、预测性结论

任意一轮失败都要先修正，再进入 Frontmatter 或发布阶段。

### Step 5: 生成 Hugo Frontmatter

生成 Frontmatter 前，必须加载以下文件：

- [../hugo-writer/SKILL.md](../hugo-writer/SKILL.md)
- [references/frontmatter-template.md](references/frontmatter-template.md)

Frontmatter 约束：
- categories 必须是 ["视频精读"]
- tags 保持 2-5 个精准名词（来源于视频核心主题和领域）
- date 必须使用生成当下的北京时间且格式完整
- description 必须是 50-100 字纯文本摘要

除非用户明确要求只输出大纲，否则写作模式默认产物应是"完整文章 + 合法 Frontmatter"。

### Step 6: 发布与落库

只有在用户明确要求发布时，才执行本步骤。

发布前必须确认：

- 文章文件路径
- 当前工作区是否就是目标仓库
- 用户是否要求同步飞书
- 用户是否要求更新发布索引

如果需要更新发布索引，按需加载 [references/publish-index-template.md](references/publish-index-template.md)。

发布结果至少回报：

- 文章文件路径
- git commit hash 或未提交状态
- 飞书文档链接或未同步原因
- publish-index 是否已更新

## 4. 输出格式

### 分析模式

- 视频信息卡（标题、频道、时长、观看量、发布时间）
- 值得写的角度
- 推荐标题与文章结构
- 证据不足项

### 写作模式

- 完整 Markdown 文章
- 合法 Hugo Frontmatter（categories: ["视频精读"]）
- 如有不确定信息，在文中显式标注边界

### 发布模式

- 写作模式全部产物
- 发布结果摘要
- 未完成项与原因

## 5. 必须与禁止

### 必须

- 先取证，再写作；禁止按视频名想象内容
- 只基于 YouTube/B站 页面、描述、评论、字幕等可验证材料写作
- 信息不足时收缩文章范围，而不是扩写空洞章节
- 默认输出中文
- 默认保留英文术语，并在首次出现时加中文括注
- 发布前完成三轮事实校验

### 禁止

- 禁止伪造观看量、点赞数、订阅数、评论内容
- 禁止把视频中没有的命令、步骤、结论写成"视频中提到"
- 禁止在未获明确指令时执行 git push、创建飞书文档、更新索引
- 禁止依赖固定机器路径；始终以当前工作区或用户指定路径为准
- 禁止为了凑结构补写"最佳实践""FAQ""延伸阅读"之类空章节

## 6. 异常处理

| 情况 | 处理 |
|------|------|
| 只有视频标题，没有链接 | 可先按标题搜索分析；无法确认时要求补充链接 |
| 视频描述很短 | 进入评论、字幕、章节补证 |
| 视频不可访问 | 告知证据不足，只能基于用户提供材料写作 |
| 视频内容明显是实验性/未完成 | 将文章改为"视频导读/内容预览"，不要写成完整教程指南 |
| 用户只要摘要 | 输出视频解读，不强制生成完整文章 |

## 8. 输出前自检

- 本文的每个关键判断是否都能回溯到视频证据
- 标题、摘要、正文是否围绕同一个主叙事
- 观看量、数据、观点、结论是否来自可验证材料
- 是否错误进入了发布模式
- Frontmatter 是否符合 Hugo 站点规范（categories: ["视频精读"]）
