# Morning Report Skill

## 定位

为 text-matrix 生产三类日报内容：AI 新闻早报 08:00、经济财经早报 08:30、AI 副业早报 09:00。

核心目标：每一条内容都必须来自过去 24 小时内、已逐条打开验证、可追溯到真实原文的快讯。

## 任务路由

收到请求后先判断范围，再决定加载哪些 references：

| 输入信号 | 任务范围 | 必须加载 |
| ------ | ------ | ------ |
| 提到 “AI 新闻”“AI 早报”“模型新闻” | 只生成 AI 新闻早报 | references/template.md + references/ai-news-sources.md |
| 提到 “财经早报”“经济早报”“市场早报” | 只生成经济财经早报 | references/template.md + references/finance-sources.md |
| 提到 “副业早报”“招聘早报”“赚钱机会” | 只生成 AI 副业早报 | references/template.md + references/side-hustle-sources.md |
| 只说 “做早报”“写早报”“morning report” | 默认生成 3 份早报 | references/template.md；随后按早报类型逐份加载对应来源文件 |

若用户没有指定日期，默认使用今天；若用户明确指定日期，仍然只采集该日期回溯 24 小时窗口内的内容。

## 按需加载

- 每次生成前都先读取 references/template.md。
- 每次只加载当前早报对应的一个来源文件，不要把 3 份来源说明一次性全部预加载。
- 生成单份早报时，不要加载另外两类来源文件。

## 执行流程

1. 识别要生成的早报类型与日期。
2. 打开对应来源站点，先收集候选条目，再逐条核验发布时间、正文主题和最终链接。
3. 删除不合格条目：超过 24 小时、没有稳定原文页、正文与标题不匹配、信息密度不足、和主题无关。
4. 依据模板整理分组，给每条内容写 2 到 3 句实质性摘要；摘要只能来自实际浏览到的信息。
5. 生成符合 Hugo 的 Markdown，输出到 content/posts/news/。
6. 输出前完成自检；任一检查失败都要回到上一步修正。

## 硬约束

### 时间窗口

- 只保留过去 24 小时内发布的内容。
- 每条都必须核实发布时间，不能按栏目页更新时间替代。
- 无法确认发布时间的条目直接丢弃。

### 链接与验真

- 每条必须包含原文字段，格式固定为 [原文](url)。
- 链接必须来自实际打开后的最终文章页，禁止使用列表页、搜索页、猜测 URL 或占位链接。
- 发布前必须逐条复查链接可访问且内容匹配。
- V2EX 帖子必须打开帖子页验证标题、正文和发布者，不能只看列表标题。

### 内容质量

- 摘要必须是信息提炼，不是改写标题，也不是泛泛而谈。
- 优先保留高信息密度条目：融资、财报、产品发布、模型更新、监管政策、市场数据、招聘需求、真实变现案例。
- 同主题重复稿只保留信息最完整的一条，必要时合并背景进摘要。

### 来源覆盖

- AI 新闻早报：至少覆盖 3 个不同来源，且至少包含 2 个中文来源和 1 个海外来源。
- 经济财经早报：至少覆盖 2 个核心财经来源；涉及市场行情时必须写出关键数字。
- AI 副业早报：至少覆盖 3 个不同来源，且至少包含 1 个社区来源和 1 个内容来源。

### Hugo frontmatter

所有早报都必须使用快讯格式，且字段完整：

- title：中文标题，带日期
- date：完整北京时间，格式 YYYY-MM-DDTHH:MM:SS+08:00
- slug：使用固定模式，不自行发明
- description：50 到 100 字纯文本摘要
- draft: false
- categories: ["行业快讯"]
- tags：2 到 5 个精准名词
- hiddenFromHomePage: true

固定 slug 规则：

- AI 新闻早报：ai-morning-news-YYYY-MM-DD
- 经济财经早报：financial-morning-news-YYYY-MM-DD
- AI 副业早报：ai-side-hustle-morning-YYYY-MM-DD

## 输出要求

- 输出目录：content/posts/news/
- 使用 references/template.md 中对应的模板骨架。
- 每条都必须独立包含标题、来源、原文、摘要；副业类额外要求发布者与标签。
- 文末数据来源汇总与正文实际引用必须一致。

## 禁止项

- 禁止编造新闻、摘要、发布者、链接、发布时间。
- 禁止把栏目页、聚合页、快讯流入口当作原文链接。
- 禁止因为来源不够就硬凑低质量内容。
- 禁止输出裸 URL、失效链接或“见原文”这类无链接文本。
- 禁止修改 slug 命名模式。
- 禁止跳过 V2EX 验真。

## 输出前自检

- 时间窗口是否全部满足过去 24 小时。
- 每条是否都打开过原文页并核对过标题与正文。
- 原文字段是否全部使用 [原文](url)。
- frontmatter 是否包含 draft: false 和 hiddenFromHomePage: true。
- slug 是否符合固定模式。
- 标签是否为 2 到 5 个具体名词，而不是空泛词。
- 数据来源汇总与正文实际引用是否一致。
- 占位文本、示例 URL、示例用户名是否已全部替换。

## 异常处理

| 情况 | 处理 |
| ------ | ------ |
| 当天高质量条目不足 | 保持少而准，不用凑数；在汇报里说明来源不足 |
| 原站没有稳定文章页 | 丢弃该条，换来源 |
| 列表页与正文信息冲突 | 以正文页为准；冲突无法消解则丢弃 |
| 用户只要草稿，不要求发布 | 仍按同样校验标准生成 Markdown，但不执行发布动作 |
