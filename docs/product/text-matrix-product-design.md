# Text Matrix 产品设计文档

## 文档信息

- 文档名称：Text Matrix 产品设计文档
- 适用范围：站点产品定位、信息架构、内容产品化、增长与转化设计
- 当前版本：V1
- 更新时间：2026-05-18
- 对应项目：`text-matrix`

## 1. 产品概述

### 1.1 产品名称

Text Matrix

### 1.2 产品一句话定位

Text Matrix 是一个面向中文开发者的 AI Agent、AI 开发工作流与开源工具知识库。

### 1.3 产品本质

Text Matrix 不是一个单纯的博客，也不是一个只追热点的 AI 资讯站。  
它的产品目标，是把高时效内容、深度技术内容、学习路径和工具判断组织成一个可检索、可复用、可持续更新的中文内容系统。

从产品形态上看，它更接近以下三者的结合：

- 技术媒体：持续覆盖 AI Agent、Coding Agent、开源工具与行业变化
- 知识库：通过专题页、分类页、搜索与内链把内容串成体系
- 内容产品：通过首页、专题页、文章页和转化入口形成稳定的用户路径

### 1.4 当前产品阶段判断

当前站点已经完成了从“内容仓库”到“初步产品化内容站”的第一阶段转换，主要特征包括：

- 首页已经不再是纯时间流，而是带有定位、分流、专题入口和 CTA 的入口页
- 已有 3 个核心专题页：
  - `/ai-agent/`
  - `/coding-agent/`
  - `/open-source-ai-tools/`
- 文章页已经补齐继续阅读与基础转化区块
- 搜索页、分类页、标签页和栏目页已具备探索型入口
- 发布流程已经加入未来时间校验与关键页面校验

但从更完整的产品视角看，当前仍处于 `内容系统化早中期`，距离“成熟内容产品”还缺少以下几层能力：

- 更清晰的用户分层与首屏分发策略
- 更明确的订阅与沉淀路径
- 更稳定的专题体系扩张机制
- 更可量化的内容与产品指标框架

## 2. 产品目标

## 2.1 总目标

把 Text Matrix 从一个高质量技术内容站，进一步推进为一个围绕 AI Agent 与开发工作流的中文知识入口。

## 2.2 分阶段目标

### 近期目标

- 强化首页与专题页的导航能力
- 让新用户更快理解站点定位与阅读入口
- 提高从首页进入专题页、从专题页进入文章页的效率
- 降低快讯内容对主品牌心智的稀释

### 中期目标

- 建立专题页驱动的内容结构，而不是单篇文章驱动
- 形成可复用的“内容发现 -> 系统阅读 -> 转化”路径
- 提高站内深度阅读率与多页访问率

### 长期目标

- 把站点沉淀成中文 AI Agent / Coding Agent / 工具工作流领域的稳定知识入口
- 在内容资产之外，形成方法论资产和工作流资产
- 为订阅、合作、咨询、产品化服务留出明确接口

## 3. 用户与场景

## 3.1 核心用户

### A 类用户：开发者

特征：

- 对 AI 工具和开发效率提升有较强兴趣
- 愿意读中长篇技术内容
- 更关注工具边界、工作流和工程实践

主要需求：

- 快速判断某个工具是否值得用
- 理解 Agent、MCP、Skills、Browser automation 等概念之间的关系
- 找到更稳定、更长期的开发工作流

### B 类用户：独立开发者 / 技术创作者

特征：

- 更关注工具选择、工作流搭建、效率提升和长期积累
- 除技术深度外，也会关心商业化、内容生产与方法沉淀

主要需求：

- 选择合适的 AI 工具栈
- 理解如何把 AI 工具接进自己的产出系统
- 获得专题化、方法论化内容，而不是零散教程

### C 类用户：AI 工具重度使用者

特征：

- 已经在使用 Claude Code、Codex、Cline、Roo Code 或类似产品
- 对 AI Agent 和 Coding Agent 的实践细节更敏感

主要需求：

- 更深的对比、选型与边界判断
- 更贴近真实项目的工作流经验
- 更高质量的参考文章和工具索引

## 3.2 次级用户

- 对 AI Agent 感兴趣的技术学习者
- 对新工具、新开源项目敏感的行业观察者
- 寻找合作、内容共创或产品交流入口的人

## 3.3 核心使用场景

### 场景 1：第一次进入站点

用户需要在 5 到 15 秒内理解：

- 这是一个什么站
- 最值得看的是什么
- 我应该从哪里开始

### 场景 2：围绕某个主题系统学习

用户不是只想看一篇文章，而是想理解一整条内容线，例如：

- AI Agent
- Coding Agent
- 开源 AI 工具

此时专题页应承担主要导航作用。

### 场景 3：读完一篇文章后继续探索

用户已经被单篇文章吸引，需要继续向下阅读时：

- 进入相关专题
- 进入相关文章
- 进入栏目页或搜索页

### 场景 4：有明确问题或工具名称时快速查找

用户并不需要首页引导，而需要：

- 搜索
- 标签
- 分类
- 专题页中的结构化入口

### 场景 5：想建立长期关注关系

用户已经认可站点价值，希望：

- 了解站点定位
- 找到联系入口
- 获得后续更新方式

## 4. 产品定位与价值主张

## 4.1 不做什么

为了让产品方向更清晰，Text Matrix 明确不以以下形态为主要目标：

- 不做泛 AI 新闻站
- 不做纯个人日记式博客
- 不做只追热点、不沉淀知识结构的内容流
- 不做只依赖单篇爆文的流量站

## 4.2 核心价值主张

Text Matrix 提供的核心价值可以概括为三层：

### 第一层：内容筛选

帮助用户判断：

- 什么值得看
- 什么值得学
- 什么值得落地

### 第二层：结构化组织

帮助用户在“文章很多”的情况下仍然能快速建立路径感，而不是被时间流淹没。

### 第三层：长期复用

站点的终极价值不只是阅读，而是把知识变成：

- 可搜索
- 可引用
- 可复习
- 可迁移到真实工作流中的内容资产

## 5. 产品结构设计

## 5.1 整体产品结构

当前产品推荐的核心结构如下：

```text
首页
├─ 专题入口
│  ├─ AI Agent
│  ├─ Coding Agent
│  └─ 开源 AI 工具
├─ 深度内容入口
│  ├─ 技术笔记
│  └─ 视频精读
├─ 时效内容入口
│  └─ 行业快讯
└─ 辅助入口
   ├─ 搜索
   ├─ 关于
   └─ 联系

专题页
├─ 适合谁看
├─ 先看哪几篇
├─ 读完会得到什么
├─ 系统阅读路径
└─ 跳转到相关文章 / 其他专题

文章页
├─ 正文
├─ 继续阅读
├─ 专题入口
└─ 转化入口
```

## 5.2 核心页面层级

### 一级页面

- 首页 `/`
- About `/about/`
- Contact `/contact/`
- Search `/search/`

### 二级页面

- 核心专题页
- 分类页
- 标签页

### 三级页面

- 单篇文章页

## 5.3 内容层级原则

内容的组织不应再完全依赖“最新发布顺序”，而应按用户目标重组：

- 首页负责第一次进入时的分流与理解
- 专题页负责系统学习和目标导向阅读
- 文章页负责深度消费和继续探索
- 搜索/分类/标签页负责问题导向的查找

## 6. 页面职责设计

## 6.1 首页职责

首页不是“展示所有内容”，而是承担 4 个具体职责：

### 1. 解释站点定位

让用户快速知道：

- 这站在写什么
- 和其他 AI 内容站有什么不同

### 2. 做首次分流

帮助不同类型用户判断应该先进入哪一条线：

- 学概念
- 学工作流
- 选工具

### 3. 做品牌强化

首页需要不断强调：

- AI Agent
- 开发工作流
- 开源工具

这是站点的主品牌，而不是所有栏目平均用力。

### 4. 提供轻转化入口

首页不需要一上来强推订阅，但要明确提供下一步动作：

- 进入专题
- 搜索主题
- 查看关于
- 联系合作

## 6.2 专题页职责

专题页是当前站点最关键的产品资产，应承担以下职责：

### 1. 降低信息散乱感

面对大量相关文章，专题页要替用户做第一轮路径设计。

### 2. 承担系统学习入口

用户可以通过专题页完成：

- 认知建立
- 起步阅读
- 进阶跳转

### 3. 承担内容索引功能

专题页不仅是“导言页”，也应逐步变成特定主题的长期入口。

### 4. 承担跨页面桥接功能

专题页应该把用户继续送往：

- 相关文章
- 其他相关专题
- 联系与转化入口

## 6.3 文章页职责

文章页是内容消费主阵地，但不应是阅读终点。

文章页主要承担：

- 提供完整内容价值
- 提供专题推荐与相关文章
- 引导用户继续进入更大的知识路径

## 6.4 搜索页职责

搜索页不是一个附属功能页，而是问题导向用户的主入口之一。

它的职责包括：

- 支持明确问题检索
- 承接首页与专题页无法完全覆盖的长尾内容
- 与标签页、专题页形成互补

## 6.5 分类页与标签页职责

分类页负责较稳定的内容组织。  
标签页负责更灵活的问题聚合。

它们应该提供的不是纯归档，而是：

- 主题解释
- 内容入口
- 继续探索路径

## 7. 用户路径设计

## 7.1 核心路径一：首页 -> 专题页 -> 文章页

适用于首次进入的新用户。

目标：

- 先理解站点
- 再选择主题
- 再进入具体内容

设计要求：

- 首页的专题入口足够明确
- 专题页能迅速给出“从哪里开始”
- 文章页能继续回流到专题体系

## 7.2 核心路径二：搜索 / 外部流量 -> 文章页 -> 专题页

适用于从搜索引擎、社交平台、收藏链接进入的用户。

目标：

- 单篇文章先完成价值交付
- 再引导用户发现更大的内容结构

设计要求：

- 文章页必须有清晰的专题推荐与继续阅读
- 文章页底部 CTA 不能只停留在“返回首页”

## 7.3 核心路径三：首页 -> 分类页 / 搜索页 -> 文章页

适用于已有方向但没有精确主题的用户。

目标：

- 通过栏目或搜索缩小范围
- 再进入单篇或专题

## 7.4 核心路径四：专题页 -> Contact / About

适用于愿意进一步建立关系的用户。

目标：

- 从内容消费进入品牌理解
- 从品牌理解进入转化或联系

## 8. 转化设计

## 8.1 当前转化阶段判断

当前更适合做 `轻转化 + 中转化`，不建议过早做过强的商业动作。

原因：

- 站点主价值仍然是内容结构和主题权威性
- 用户更需要建立信任，而不是立刻被要求订阅或购买

## 8.2 转化层级

### 轻转化

- 进入专题页
- 继续阅读
- 使用搜索
- 浏览分类页

### 中转化

- 查看 About
- 查看 Contact
- 通过联系页建立进一步连接

### 强转化

后续可逐步扩展到：

- 邮件订阅
- 社群
- 咨询合作
- 产品化服务

但这些动作应建立在内容结构已经成熟的前提上。

## 8.3 当前最推荐的转化策略

### 首页

- 突出专题入口
- 保留 Contact / About / Search 作为轻量行动入口

### 专题页

- 持续强化“下一步应该去哪”
- 增加专题之间的跳转矩阵
- 让专题页承担更多“持续关注某主题”的语义

### 文章页

- 强化继续阅读
- 强化专题回流
- 强化联系入口

## 9. 信息架构原则

## 9.1 一级原则

- 先按用户目标组织，再按发布时间组织
- 先按主题建立路径，再按单篇沉淀内容
- 先做结构清晰，再做内容扩张

## 9.2 内容分发原则

### 技术笔记

- 主承担长期搜索与知识沉淀
- 应获得首页和专题页更高权重

### 视频精读

- 承担高价值外部内容的结构化转译
- 适合作为理解新主题的过渡入口

### 行业快讯

- 承担覆盖与触达
- 不应长期挤占首页主心智

### 辅助栏目

- 用于品牌表达和边缘主题补充
- 不承担主增长职责

## 10. 当前产品问题

## 10.1 已被缓解的问题

- 首页过去过于时间流导向，已初步改善
- 专题内容缺少正式入口，已建立 3 个核心专题页
- 文章页阅读终点过强，已加入继续阅读与转化入口
- 发布流程容易缺页，已加入校验

## 10.2 仍待解决的问题

### 1. 首页与专题页联动仍然偏浅

虽然已有专题入口，但首页尚未完全形成“专题即核心导航”的结构。

### 2. 专题页更像高质量导览页，离成熟专题中心还有距离

后续还需要继续沉淀：

- 更强的精选文章卡片
- 更明确的按目标阅读路径
- 更清晰的专题互链

### 3. 订阅与沉淀路径不够明确

当前已有联系入口，但还没有更明确的“持续跟进这个主题”的产品语义。

### 4. 缺少面向产品的指标定义

目前站点更像内容运营系统，尚未形成一套更明确的产品指标框架。

## 11. 指标设计

## 11.1 核心产品指标

推荐优先关注以下指标：

- 首页进入专题页的点击率
- 专题页进入文章页的点击率
- 单次会话浏览页数
- 专题页平均停留时长
- 文章页到专题页的回流率
- 搜索页使用率
- Contact / About 访问率

## 11.2 内容指标

- 技术笔记的搜索流量占比
- 快讯内容的入口流量占比
- 专题页承接的内容分布
- 高价值文章的长期访问曲线

## 11.3 后续可扩展指标

- 专题页的收藏 / 回访表现
- 订阅转化率
- 合作咨询入口点击率

## 12. 产品路线图

## 12.1 P0：已完成或接近完成

- 首页产品化入口改造
- 3 个核心专题页上线
- 文章页继续阅读与转化入口
- 搜索 / 分类 / 标签页探索能力增强
- 发布系统关键护栏补齐

## 12.2 P1：下一阶段重点

### 1. 首页与专题页联动增强

目标：

- 让专题成为首页核心导航结构

动作：

- 让首页专题卡片更前置
- 给专题入口加更明确的人群标签
- 给专题入口加“先读哪篇”

### 2. 专题页继续增强

目标：

- 从导览页走向专题中心

动作：

- 增加精选文章卡片区
- 增加专题间互链矩阵
- 增加更明确的专题型 CTA

### 3. 上线后专题页验收清单

目标：

- 验证当前专题页在真实访问中的可用性和路径清晰度

动作：

- 验证移动端表现
- 验证首页到专题页的点击路径
- 验证专题页到文章页的阅读顺序是否自然

## 12.3 P2：中期能力建设

- 建立更明确的订阅或更新关系
- 引入更系统的专题页扩张机制
- 建立产品指标面板或最基础埋点框架

## 12.4 P3：长期方向

- 把内容生产系统产品化
- 形成围绕 AI Agent / Coding Agent 的更强品牌资产
- 将内容站扩展成更完整的知识产品入口

## 13. 验收标准

一份“产品层面可接受”的版本，至少应满足：

- 用户首次进入首页时能快速理解站点定位
- 用户能在首页快速判断自己该进入哪一个专题
- 用户进入专题页后能在 10 到 20 秒内知道从哪开始读
- 用户读完文章后不会断路，而是能自然进入专题页或相关文章
- 站点发布时，首页、专题页、搜索页与关键入口不会静默缺失

## 14. 风险与边界

## 14.1 主要风险

### 1. 继续扩内容而不扩结构

如果只增加文章数量而不继续加强专题与路径设计，站点会重新回到“内容多但不易消费”的状态。

### 2. 快讯占比过高

快讯能带来更新感，但如果占据过多注意力，会削弱主品牌的长期心智。

### 3. 转化动作过早过强

如果在用户尚未建立信任前就过强推动订阅或合作，反而会削弱内容体验。

## 14.2 当前边界

本文档主要聚焦：

- 站点内容产品层
- 信息架构与页面职责
- 用户路径与转化设计

不覆盖：

- 编辑团队流程细则
- 完整 SEO 运营方案
- 商业化方案细则
- 数据埋点实施技术细节

## 15. 结论

Text Matrix 当前最正确的产品方向，不是继续把自己做成“更大的内容站”，而是继续把自己做成“更清晰的主题入口”。

后续所有重要改进，都应服务于同一个目标：

- 让用户更快理解站点
- 让用户更快进入正确专题
- 让用户更容易沿着主题持续阅读
- 让内容从一次性消费，逐步变成长期可复用的知识资产

如果后续产品演进保持这个方向，Text Matrix 的差异化不会来自“文章数量更多”，而会来自“它比别的站更会组织 AI Agent 与开发工作流内容”。
