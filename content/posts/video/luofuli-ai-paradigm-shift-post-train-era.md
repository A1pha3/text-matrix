## 前言

2026年，AI战争的第二幕已然拉开。

第一幕是Pre-train（预训练）主导的Chat时代，以GPT-4、Claude 3等为代表模型，核心竞争在于"谁能训练出更强的基座模型"。第二幕是Post-train（后训练）主导的Agent时代，以Claude Opus 4.6、OpenClaw等技术变量为触发点，核心竞争在于"谁能训练出更强大的Agent能力"。

这一次技术范式的转移，对整个AI产业提出了全新的命题：**如何在新时代下做出技术押注？如何重新分配算力资源？组织如何重组？**

带着这些疑问，访谈了人工智能研究员罗福莉。

## Pre-train到Post-train的范式转移

### 旧时代：Pre-train为核心的卡分配

在Chat时代，算力资源的分配呈现极不平衡的状态：

| 用途 | 卡分配比例 |
|------|------------|
| 研究 | 3 |
| Pre-train | 5 |
| Post-train | 1 |

罗福莉透露，这是一个"非常夸张"的比例——Pre-train占用了绝大部分算力，而Post-train几乎被忽视。

### 新时代：1:1的算力分配

进入Agent时代后，这一比例发生了根本性变化：

> 一个非常合理的用卡比例可能是**3:1:1**。
> 
> 预训练跟后训练一个比例，这是今年可能发生的很大变化。顶尖团队应该都是**1:1**了。

这意味着：

1. **Pre-train和Post-train同等重要**
2. **研究的卡量相对固定**（3份）
3. **资源正在从Pre-train向Post-train转移**

### 为什么Post-train突然变得重要？

核心原因在于：**Agent时代的竞争焦点已经转移**

在Chat时代，核心问题是"模型够不够聪明"——这主要由Pre-train决定。

在Agent时代，核心问题是"模型能不能完成任务"——这主要由Post-train（尤其是RL Scaling）决定。

> 现在至少跟23年要去追平Pre-train的差距一样，大家很all in，要去做好Agent的Post-train。更具体说，是在Agent上怎么做好**RL的scaling**。

## 群体智能：Agent的协作范式

### 从单Agent到多Agent系统

Agent时代的一个核心变化是：**从追求单个模型的智能，转向追求多个Agent的协作智能**。

群体智能（Collective Intelligence）理论认为，当多个智能体通过协作、竞争和通信形成整体时，可以涌现出超越单个智能体的能力。这正是当下Agent框架演进的方向。

### Agent框架的核心要素

一个成熟的Agent系统需要具备：

| 要素 | 说明 |
|------|------|
| **规划能力** | 将复杂任务分解为可执行的子步骤 |
| **记忆机制** | 跨对话保持上下文和历史信息 |
| **工具使用** | 调用外部API、搜索、代码执行等 |
| **协作通信** | 多Agent之间的信息交换与任务协调 |

### 为什么群体智能突然重要？

当单Agent能力达到瓶颈时，群体协作成为新的突破方向：
- 专业化分工：不同Agent专注于不同能力
- 知识蒸馏：群体知识可以压缩到单个模型
- 容错性：部分Agent失败不影响整体任务完成

> 群体智能的Scaling，可能是下一个Scaling Law。

## Agent的自进化与自迭代

### 什么是Agent的自进化？

自进化（Self-Evolution）是指Agent能够：
1. **从反馈中学习**：根据执行结果调整策略
2. **自我诊断**：识别自身能力边界
3. **主动扩展**：在边界处主动寻求突破

### 自迭代的技术基础

| 技术 | 作用 |
|------|------|
| **RL + Rollout** | 通过试错学习最优策略 |
| **Meta-Learning** | 学会如何学习 |
| **Self-Reflection** | 反思失败并改进 |
| **Curriculum Learning** | 从简单到复杂的渐进学习 |

## 训练细节与成本：Scaling的经济学

### 算力成本的结构

大模型训练的算力成本可以分解为：

| 成本项 | 占比 | 说明 |
|--------|------|------|
| **Pre-train** | 60-70% | 基础模型训练 |
| **Post-train** | 15-20% | SFT、RLHF等 |
| **RL Scaling** | 10-15% | Agent能力训练 |
| **研究实验** | 5-10% | 探索性实验 |

### 从5:1到1:1的算力再分配

罗福莉提到的算力分配变化：

```
旧时代（Chat时代）：
- Pre-train : Post-train = 5:1

新时代（Agent时代）：
- Pre-train : Post-train = 1:1
```

这背后的逻辑是：
1. **Pre-train的边际收益递减**：更大的模型不带来质的飞跃
2. **Post-train的边际收益递增**：更好的Post-train直接转化为Agent能力
3. **竞争焦点的转移**：从"训练更强的模型"到"训练更有用的模型"

### 训练成本的实际数字

业界估算（仅供参考）：

| 训练阶段 | 成本量级 | 典型耗时 |
|---------|---------|----------|
| 1T tokens | ~$100万 | 1-2周 |
| 10T tokens | ~$1000万 | 2-3月 |
| 100T tokens | ~$1亿 | 半年+ |
| RL Scaling | $100-$500万 | 持续迭代 |

## 另类架构：非Transformer的探索

### Transformer的天花板？

Transformer架构统治AI领域已超过7年。但随着模型规模扩大，其局限性也开始显现：
- **O(n²)注意力复杂度**：长上下文计算成本高昂
- **对重复模式的偏好**：容易陷入思维定式
- **难以处理结构化知识**：对因果推理有固有缺陷

### 另类架构的方向

| 架构方向 | 代表工作 | 核心优势 |
|---------|---------|---------|
| **State Space Model** | Mamba | 线性复杂度的序列建模 |
| **Mixture of Experts** | Mixtral | 稀疏激活降低计算成本 |
| **Recurrent接口** | RNN/GRU变体 | 更长的有效上下文 |
| **神经符号混合** | AlphaCode等 | 结合神经网络与符号推理 |
| **扩散模型** | DDPM用于语言 | 另一种生成范式 |

### 小米的架构选择

> 到底去Scaling模型的参数量，还是去Scaling什么东西？

这个问题在架构层面同样适用：
- **宽度 vs 深度**：更宽的Attention还是更深的网络？
- **专家混合 vs 稠密**：稀疏激活还是稠密计算？
- **位置编码 vs 无位置**：RoPE、ALiBi还是彻底的位置无关？

## AI没有生存危机

### "生存危机"论的误区

当下关于AI对人类的威胁，主要有两派：

| 论调 | 代表人物/观点 |
|------|-------------|
| **末日论** | AI将取代人类，导致文明崩溃 |
| **乌托邦论** | AI将解决所有问题，人类获得永生 |

罗福莉的观点更接近**实用主义**：

> AI是一种工具，它在某些任务上已经超越人类，在其他任务上还远不如人类。

### AI真正的局限性

| 局限 | 说明 |
|------|------|
| **常识推理** | 缺乏物理世界的直觉理解 |
| **长期规划** | 难以进行跨越多年的规划 |
| **价值判断** | 依赖人类定义目标和价值观 |
| **创造力** | 只能重组已有关知识，无法真正创新 |

### 人类何去何从

当AI内化人类知识后，人类的新角色：

1. **价值定义者**：告诉AI要追求什么目标
2. **边界守护者**：确保AI在安全边界内运行
3. **创新引领者**：探索未知领域，开辟新方向
4. **意义赋予者**：为行动赋予意义和目的
