## 先给结论

如果你时间有限，可以先记住下面 8 点：

1. 这场访谈真正的中心句，不是“ AGI 两年内会不会实现”，而是大模型竞争已经从 Chat 时代进入 Agent 时代，竞争焦点从 Pre-train 转向 Post-train。
2. “ Anthropic 的路径是正确的，这是当下共识”之所以重要，不是因为一句站队，而是因为行业对 Agent 场景里 Context 工程、 Harness 和长程任务完成度的理解发生了变化。
3. 在罗福莉的判断里，国内团队在 Pre-train 上的代差已经非常接近，新的差距会出现在 Agent 的后训练、 RL Scaling 和框架适配能力上。
4. Chat 时代常见的研究、 Pre-train、 Post-train 用卡比例可以是 3:5:1；到了 Agent 时代，更合理的比例会走向 3:1:1，顶尖团队的预训练和后训练甚至会接近 1:1。
5. OpenClaw 被反复提及，不是因为它已经解决了一切，而是因为它把一个行业盲点暴露出来：真正决定 Agent 上限的，不只是模型本身，还有那层厚重的中间框架。
6. 1T 是 Agent 时代的入场券，但这里更接近“ 1T 参数级基座模型”的意思，不是“ 1T tokens 就够了”；真正难的是下一步到底该 scaling 什么、在哪里 scaling。
7. 2026 年更像“生产力加速变革之年”，工作方式会先被颠覆，生活方式反而未必最先改变。
8. 这场访谈最冷的一句话不是 AGI，而是“每天在否认昨天的自己”。它说明，研究范式、组织方式和个人能力结构都在被同时改写。

## 这场访谈真正的主线：竞争中心已经变了

访谈里最值得反复咀嚼的一句判断是：

> **Anthropic 的路径是正确的，这是当下共识。**

如果只看字面，这像一句行业站队；但放回 2026 年的上下文里，它的分量很重。因为它意味着，很多团队对 Claude 系列成功原因的理解，已经从“模型本身更强”转向“模型、框架、上下文工程和后训练一起构成了更高的任务完成度”。

36kr 在整理里专门补了一层解释：过去不少人以为 Claude 做那么多 Context 工程，是因为模型结构不够先进，或者是为了成本妥协；但现在回头看，那些设计更像是在为 Agent 场景里的整体任务完成度服务。这个认知变化，恰好就是“为什么竞争主线从 Pre-train 迁移到 Post-train”的关键。

罗福莉在视频简介里的原话也很明确：

> 现在大家在 Pre-train 上的代差是基本没有的，或者说非常接近。

36kr 的整理又把这层意思进一步量化成“中美模型代差只有 2 到 3 个月”。无论你接受哪种口径，指向的都是同一件事：**仅仅把基座模型再做大，已经不太足以构成新的决定性优势。**

在 Chat 时代，大家主要比谁更聪明；在 Agent 时代，大家开始比谁更能完成任务。前者主要由 Pre-train 决定，后者则更依赖 Post-train、 RL Scaling、框架适配、长上下文管理和错误恢复能力。换句话说，行业竞争的“评分函数”本身已经变了。

## 为什么 Post-train 突然成了主战场

罗福莉在访谈中有一句非常具体的判断：

> 现在至少跟 23 年要去追平 Pre-train 的差距一样，大家很 all in，要去做好 Agent 的 Post-train。更具体说，是在 Agent 上怎么做好 RL 的 scaling。

这句话的重点，不只是“后训练更重要了”，而是后训练的角色已经从过去的末端补丁，变成决定 Agent 是否可用的核心环节。

从她的说法看，Agent 时代至少发生了 3 个变化：

1. 模型要面对的任务不再只是问答，而是长程、多步、可失败、可恢复的真实执行过程。
2. 系统的中心不再只是一个 Rollout 推理引擎，而是一个以 Agent 为核心、带工具、带记忆、带状态的复杂系统。
3. 团队要解决的问题，也从“把模型训得更强”变成“把模型放进真实系统里后，怎样让它持续做对”。

访谈原话里还有一句配套判断：

> 系统从“以 Rollout 推理引擎为核心”，转变为“以 Agent 为核心”的一个更复杂系统。

这句话可以当作整场访谈的技术底色。它意味着，今天讨论 AI 竞争，已经不能只盯着模型参数、训练 tokens 或单轮 Benchmark。真正拉开差距的地方，越来越多出现在模型与环境交界的那层系统里。

## 算力怎么重排：从 3:5:1 到 3:1:1

这场访谈里最可操作、也最容易被低估的内容，是罗福莉给出的算力配置判断。她提到，至少在 Chat 时代，研究、 Pre-train、 Post-train 的用卡比例可能是一个非常夸张的 3:5:1；而进入 Agent 时代后，更合理的比例会变成 3:1:1，顶尖团队的 Pre-train 与 Post-train 甚至已经接近 1:1。

| 时代 | 研究 | Pre-train | Post-train |
| ---- | ---- | --------- | ---------- |
| Chat 时代 | 3 | 5 | 1 |
| Agent 时代 | 3 | 1 | 1 |

这组数字真正说明的，不是“ Pre-train 不重要了”，而是**边际收益和系统瓶颈一起改了位置**。

在旧范式里，更强的基座模型可以直接带来更强的体验，所以 Pre-train 吃掉绝大多数卡是合理的。到了新范式里，更强的模型如果不能稳定地跑长程任务、理解复杂上下文、在多步执行里自我修正，它的能力就很难真正转化为生产力。这时，Post-train 就不再是锦上添花，而是把模型能力落地成 Agent 能力的关键通道。

36kr 整理稿里还有一个很有价值的补充：随着 Agent 进入研究流程，一个 idea 从写代码到设计评估，可能从一两周压缩到一两个小时。效率提升之后，研究卡、训练卡、推理卡不一定更宽松，反而可能更紧。因为能并行验证的想法突然变多了，整个系统的吞吐压力会上来。

所以，罗福莉谈“卡怎么分”，本质上不是在谈一个静态预算表，而是在谈：**当研究速度和系统复杂度一起上升时，资源瓶颈会往哪里转移。**

## 组织为什么也得跟着改

技术路线一旦变，组织方式通常也得跟着重写。

罗福莉在访谈里给出的一个很重要的组织判断是：做后训练需要 diversity（多样性），让预训练的人做后训练，是一个很好的补充。这句话看似简单，实际上是在拆旧的分工边界。

在旧的团队分法里， Pre-train、 Post-train、 Infra、产品往往是各自成岛的。新范式下，这样的壁垒会变成拖慢反应速度的来源。因为 Agent 场景的问题，往往不是单一学科问题：它同时涉及模型行为、上下文编排、工具调用、 RL 反馈、评估标准和系统基础设施。只让“后训练团队”单独解决，容易出现局部最优。

访谈后半段谈到 MiMo 团队时，罗福莉还给出了另一个更有争议、但也更有启发的判断：环境比经验更重要。她不太迷信既有“大模型经验”，更看重人在好环境里能否迅速学会新范式里的工作方式。结合她提到的平权组织、较少职级分层、快速调试与联合排障，这场访谈给出的其实不是一个“公司八卦”，而是一种组织信号：**范式切换期，静态经验的价值可能下降，适应速度和高质量环境的价值会上升。**

## OpenClaw 为什么会在这场访谈里反复出现

如果只看标题，很多人会以为这场访谈是在谈某个具体产品。但实际上， OpenClaw 在这里更像一个行业压力测试器。

36kr 对这一点的提炼很到位：罗福莉把 OpenClaw 看作“划时代的 Agent 框架”，不是因为它只是一个好用的产品，而是因为它定义了系统怎样跟模型沟通。真正重要的，不是最表面的交互层，而是那套厚重的中间层：记忆系统、消息通道、心跳任务、定时任务、 Workflow、 Multi-Agent 逻辑，以及如何把模型自己拿不到的环境信息补进来。

这也是她为什么特别看重开源。对于黑盒产品，外界最多只能感受到“效果不错”；对于开源框架，用户却可以直接改 memory、改 workflow、改 Multi-Agent 逻辑，甚至借助更强模型去重写框架本身。这种“原生可操纵性”，才是 OpenClaw 在访谈里被反复拿来讨论的原因。

36kr 还记录了一个很有意思的判断：当 Opus 这类顶尖模型先把框架改好之后，再切回 Sonnet、国内模型，甚至 MiMo 自己的模型，体验也会被明显拉升。这个观察很重要，因为它说明 Agent 框架的价值，不只是把顶尖模型的能力释放出来，还可能**抬高中层模型的有效上限**。

## 群体智能值得看，但不该被写成万能答案

原视频大纲里专门留了“群体智能提升 Agent 框架”这一章。这个话题容易被越写越大，所以更需要收住边界。

从现有来源看，罗福莉对群体智能的兴趣主要落在两个层面。

第一，它可以加快框架迭代。一个人改框架，速度很慢；一群人在社区或工作群里一起用、一起改、互相激发想法，几小时就可能迭代一轮。这是开源 Agent 框架非常真实的优势。

第二，它可能帮助形成更高效的分工和更便宜的执行路径。多智能体协作有机会提高速度、降低部分场景成本。

但同样要注意， 36kr 的整理里也明确写了：罗福莉并没有看到足够证据证明 Multi-Agent 已经显著提高任务完成率上限。她甚至用“现在很多还有点伪”来形容一些多智能体叙事。

所以，更稳妥的理解应该是：**群体智能目前更像框架演化的加速器，而不是已经被证实的新 Scaling Law。** 这一点恰好也是原稿最容易过度发挥的地方。

## 1T 是入场券，但真正难的是下一步怎么 scaling

访谈里另一句非常关键的话是：

> 我们不会在 1T 水平上走太久。

这句话经常被转述，但也最容易被转述错。结合视频简介和 36kr 的整理，更合理的理解是：**1T 参数级基座模型，是 Agent 时代接近顶尖水平的一张入场券。** 它说的不是“ 1T tokens 就够了”，也不是“只要有 1T 就能赢”。

真正的问题在后面：到底去 scaling 参数量，还是 scaling 其他维度？要在什么样的芯片上 scaling？怎样把长上下文、低成本和高速度同时兼顾？

这组问题之所以关键，是因为 Agent 任务和普通聊天任务不一样。它不是多塞点上下文就结束，而是要在长时间、多步骤执行中保持一致性、理解深层要求、恢复失败状态、控制成本。如果 1M 上下文很贵、很慢、很难用，那么再长的上下文也只是一个纸面参数。

这也是为什么把后续公开资料作为补充背景时，会看到一个很有意思的对应关系。以 MiMo-V2.5-Pro 为例，外部报道把它的重点放在 long-horizon coherence、 agentic post-training 和对 Agent scaffold 的适配上。这不能反向证明访谈原话，但至少说明，罗福莉在访谈里强调的方向，后来确实体现在团队的公开产品叙事里。

## 2026 年真正先被改写的，是工作方式

最抓眼球的标题当然是“ AGI 两年内到来”。但如果从工程和产业角度看，访谈里更值得重视的其实是另一层判断：工作方式会先被颠覆，而不是生活方式。

36kr 的整理把这层意思展开得很清楚。罗福莉认为，现在 AGI 进程大概走到 20% 左右，今年至少能走到 60% 到 70%；但更重要的是，首先被改写的是工作系统。原因很直接：工作直接产生生产力价值，而生活场景的重构还更依赖机器人、硬件和物理世界能力。

这里面有两个值得工程师特别留意的变化。

第一，人和 Agent 的交互层级会上移。人不再主要负责指出“这行代码错了”，而是更多负责补限制条件、澄清需求、给出业务逻辑、判断优先级。这不是人变得不重要，而是人类价值从低层执行迁移到了高层判断。

第二，很多高价值信息并不在预训练数据里。企业内部流程、团队经验、业务规则、技能清单、长期上下文，这些东西往往无法靠互联网语料自动补齐。罗福莉把这类信息看成预训练之外的另一种“阿尔法”。这也是为什么 Skills、 Memory、长上下文和 Agent 框架会一起变重要。

从这个角度看，“工作方式先被颠覆”并不是一句抽象口号，而是对数据形态和人机分工都已经开始变化的描述。

## 这场访谈最冷的一句，是“每天在否认昨天的自己”

如果说前面的内容主要在谈行业和技术，那么这句“每天在否认昨天的自己”，谈的就是研究者当下最真实的心理结构。

在访谈里，罗福莉提到她之前以为自己的工作已经足够有创造力、足够不会被 Skill 化、 Workflow 化，但后来发现 AI 竟然也能逼近这类工作，甚至可能进一步参与模型训练本身。这个冲击比“某个模型跑分更高”要深得多，因为它直接作用在研究者对自身不可替代性的判断上。

这也是为什么“环境比经验更重要”会和“每天否认昨天的自己”出现在同一场访谈里。前一句说的是组织如何应对范式切换，后一句说的是个人如何承受范式切换。两者合在一起，构成了 2026 年 AI 行业非常鲜明的一种气氛：**昨天有效的方法今天可能就不够了，而新的方法还在快速形成。**

## 这场访谈里，有 3 个地方不宜过度解读

为了避免把这篇文章读成情绪放大器，下面 3 个边界最好一开始就守住。

### 1. AGI 时间表是高强度一线研究者的判断，不是行业共识

“两年内应该能实现”当然值得重视，但它仍然是一种前瞻判断，不是已经验证完成的事实。更稳妥的读法，是把它当作对技术加速度的主观估计，而不是日历式预言。

### 2. Multi-Agent 值得跟踪，但上限提升还没有被充分证明

从现有来源看， Multi-Agent 更明确的价值在于提高迭代速度、改善某些场景的成本效率，而不是已经被证实可以系统性抬高任务成功率上限。

### 3. 开源框架很重要，但它不能替代评估、资源和工程能力

OpenClaw 这样的框架让很多能力变得可见、可改、可迭代，这非常关键；但如果没有好的评估体系、足够快的 infra、明确的约束和足够强的基座模型，开源本身也不会自动生成结果。

## 对工程师和团队来说，最值得带走的 4 个动作

如果把这场访谈转译成具体行动，我觉得至少有 4 件事值得马上做。

### 1. 不要只测聊天能力，要测长程任务完成度

如果你的评估还停留在单轮问答或几个静态 Benchmark 上，那很可能已经落后一代。更应该看的，是模型在真实 Agent 任务里的稳定性、错误恢复能力、上下文利用率和工具调用质量。

### 2. 把 Harness、 Memory 和评估系统当作核心基础设施

访谈里反复出现的一个隐含结论是：模型并不是直接裸奔进生产环境的。真正决定体验上限的，是框架怎样喂上下文、怎样存状态、怎样做约束、怎样评估失败。把这些当“外设”，通常会吃亏。

### 3. 重新审视资源预算，不要把 Post-train 当尾声

如果你还在按旧范式理解资源分配，很容易低估 Agent 时代后训练、 RL 和推理系统的重要性。新的预算表，不只是把卡从 A 栏移到 B 栏，而是重新定义“什么地方最值得花卡”。

### 4. 让团队更跨界，而不是更分层

Pre-train、 Post-train、 Infra、产品、评估这些能力，在 Agent 时代越来越难被彻底切开。让不同背景的人更早进入同一个问题空间，通常比把分工切得更细更重要。
