## 项目概览

[Dexter](https://github.com/virattt/dexter) 是一个基于LLM的自主金融研究Agent，核心能力是**将复杂的金融问题转化为清晰的分步研究计划，并使用实时市场数据执行分析**。它能像专业分析师一样思考、规划和学习，直到给出有数据支撑的结论。

**核心数据：**
- GitHub Stars：409
- 技术栈：TypeScript/Bun
- 依赖服务：OpenAI API、Financial Datasets API、Exa API（可选）
- License：MIT

**核心功能：**
- 智能任务规划：自动将复杂查询分解为结构化研究步骤
- 自主执行：选择并执行正确的工具获取金融数据
- 自我验证：检查自身工作质量并迭代优化
- 实时财务数据：接入财报（损益表、资产负债表、现金流量表）
- 安全护栏：内置循环检测和步数限制，防止失控执行

## 核心问题与解决思路

### 传统金融研究的痛点

人工做金融研究存在几个根本性问题：
1. **效率低**：需要手动查找、整理大量分散的数据源
2. **覆盖面窄**：个人精力有限，难以同时跟踪多个标的
3. **主观偏差**：容易受到情绪和先入为主观念影响
4. **更新滞后**：手动研究难以做到实时跟踪

### Dexter的解决思路

Dexter模拟专业分析师的工作方式，但以自动化形式实现：

```
用户查询 → 任务规划 → 工具选择 → 数据获取 → 自我验证 → 迭代优化 → 最终报告
```

每个环节都有AI驱动的决策和执行能力。

## 系统架构

### 技术栈

- **运行时**：Bun（v1.0+）
- **核心语言**：TypeScript
- **AI模型**：OpenAI GPT系列
- **数据源**：Financial Datasets API（收入、资产负债表、现金流量表）
- **搜索**：Exa API（可选，用于网络搜索）

### 目录结构

```
dexter/
├── src/
│   ├── agents/          # Agent核心定义
│   ├── tools/           # 外部工具集成（金融数据、网络搜索）
│   ├── workflows/       # 工作流编排
│   └── utils/           # 辅助工具
├── env.example          # 环境变量示例
├── bun.lockb            # 依赖锁定
└── README.md
```

### 核心模块

**1. 任务规划Agent**
- 接收用户金融问题
- 分解为可执行的研究步骤
- 确定所需数据类型和数据源

**2. 工具执行层**
- 财务报表API：获取收入、利润、负债等结构化数据
- 网络搜索：获取新闻、公告、分析报告
- 数据验证：交叉验证数据准确性

**3. 自我验证机制**
- 置信度评估：对每个结论评估可信度
- 交叉验证：多数据源相互印证
- 迭代优化：对低置信度结论触发重新查询

## 安装与配置

### 前置依赖

| 依赖 | 说明 | 获取方式 |
|------|------|---------|
| Bun | JavaScript运行时 | [bun.sh](https://bun.sh) |
| OpenAI API Key | GPT模型调用 | [platform.openai.com](https://platform.openai.com/api-keys) |
| Financial Datasets API | 财务报表数据 | [financialdatasets.ai](https://financialdatasets.ai) |
| Exa API Key | 网络搜索（可选） | [exa.ai](https://exa.ai) |

### 安装步骤

**1. 克隆仓库**
```bash
git clone https://github.com/virattt/dexter.git
cd dexter
```

**2. 安装依赖**
```bash
bun install
```

**3. 配置环境变量**
```bash
cp env.example .env
# 编辑.env，填入API密钥
```

`.env`示例：
```env
OPENAI_API_KEY=sk-xxxx
FINANCIAL_DATASETS_API_KEY=fd_xxxx
EXA_API_KEY=exa_xxxx  # 可选
```

**4. 启动**
```bash
bun run start
# 或开发模式
bun run dev
```

### 验证安装

```bash
# 检查环境配置
bun run validate-env

# 运行测试
bun test
```

## 使用方法

### 基础查询

在终端输入金融问题：

```bash
# 分析苹果公司的财务状况
bun run start "分析苹果公司(AAPL)最近一季度的财务表现"

# 研究特斯拉的竞争优势
bun run start "特斯拉的竞争优势和潜在风险有哪些"

# 行业对比分析
bun run start "比较英伟达和AMD的GPU业务表现"
```

### 输出示例

Dexter的分析输出包含：
1. **执行摘要**（Executive Summary）
2. **关键发现**（Key Findings）
3. **数据支撑**（Data Evidence）
4. **置信度评级**（Confidence Level）
5. **风险提示**（Risk Factors）

## 核心能力详解

### 1. 智能任务规划

Dexter的任务规划不是简单的关键词匹配，而是理解查询意图后生成结构化计划：

```
原始查询："苹果公司值得投资吗？"
    ↓ 意图理解
分解为：
  - [Step 1] 获取AAPL最新季度财报数据
  - [Step 2] 提取关键指标（营收、净利润、毛利率、EPS）
  - [Step 3] 与历史数据对比，分析趋势
  - [Step 4] 获取分析师评级和目标价
  - [Step 5] 汇总分析，输出投资建议摘要
```

### 2. 财务报表深度分析

Dexter接入Financial Datasets API，能获取完整的SEC filings数据：

| 数据类型 | 涵盖内容 |
|---------|---------|
| 损益表 | 营收、COGS、毛利、EBIT、净利润、EPS |
| 资产负债表 | 总资产、总负债、股东权益、现金及等价物 |
| 现金流量表 | 经营活动现金流、投资活动现金流、融资活动现金流 |

### 3. 自我验证与迭代

每个结论都会经过置信度评估：

```
如果置信度 < 0.7：
    → 触发补充查询
    → 获取更多佐证数据
    → 重新评估结论
如果置信度 >= 0.7：
    → 进入最终报告
```

### 4. 安全护栏

Dexter内置了两层安全机制：

| 护栏 | 说明 |
|------|------|
| 循环检测 | 同一查询路径超过3次自动终止，防止死循环 |
| 步数限制 | 单次研究任务最大50步，超出强制结束 |
| 错误处理 | API调用失败自动降级或跳过，不中断整体流程 |

## 适用场景

### 适合的场景
- 个股基本面研究（财报分析、估值对比）
- 行业研究（竞争格局、市场空间）
- 投资初筛（快速筛选潜在标的）
- 投研报告辅助（获取初步分析框架）

### 边界与局限
- **不提供投资建议**：Dexter输出是分析框架和数据，不是投资推荐
- **依赖数据质量**：分析准确性受限于API数据质量
- **非实时行情**：不提供秒级实时价格数据
- **模型能力限制**：复杂推理仍受LLM能力约束

- 将分析师的思维方式工程化
