# OpenSpace：让 AI Agent 自我进化的引擎

> **难度**：⭐⭐⭐⭐ | **类型**：专家设计 | **预计阅读时间**：25 分钟
> **目标读者**：AI 开发者、Agent 系统架构师、技术决策者

---

## 🎯 核心价值

在正式介绍之前，先看一组令人震撼的数据：

| 指标 | 数值 | 说明 |
|------|------|------|
| **Token 节省** | 46% | 相同任务下，Token 消耗减少近一半 |
| **收入提升** | 4.2 倍 | 在 GDPVal 基准测试中，OpenSpace Agent 收入是基线的 4.2 倍 |
| **进化 Skill 数** | 165 项 | 在 50 项任务中自主进化出的可复用 Skill |
| **质量提升** | +30pp | 比最佳基线 Agent（40.8%）高出 30 个百分点 |

这三个数字背后，是一个根本性的范式转变：**AI Agent 不再是每次任务都从零开始，而是能从真实世界的经验中持续学习和进化**。

---

## 🔴 当前 AI Agent 面临的困境

如今的 AI Agent——OpenClaw、nanobot、Claude Code、Codex、Cursor 等——能力强大，但有一个致命弱点：

> **它们从不从真实世界的经验中学习、适应和进化——更不用说相互之间的共享了。**

### 三大核心问题

| 问题 | 具体表现 |
|------|----------|
| **❌ 大量 Token 浪费** | 如何复用成功的任务模式，而非每次都从零推理、烧掉大量 Token？ |
| **❌ 重复犯下高代价错误** | 如何在 Agent 之间共享解决方案，而非反复进行同样昂贵的探索和犯同样的错？ |
| **❌ Skill 质量差且不可靠** | 当工具和 API 持续演变时，如何保证 Skill 的可靠性——同时确保社区贡献的 Skill 达到严格的质量标准？ |

这些问题揭示了一个深层矛盾：**Agent 越强大，每次任务消耗越多；但强大的推理能力并没有被积累下来，下次任务依然从零开始。**

---

## 🚀 OpenSpace 是什么？

**OpenSpace** 是一个自我进化引擎，让每一次任务都能使每个 Agent 变得更聪明、更高效。

它的核心思想很简单：**把「如何完成任务」的知识变成可积累、可复用、可进化的 Skill。**

### 三大超能力

OpenSpace 以 Skill 的形式接入任意 Agent，并赋予其三大超能力：

### 🧬 超能力一：自我进化

Skill 能够自动学习并持续提升：

| 进化模式 | 说明 |
|----------|------|
| **✅ 自动修复（AUTO-FIX）** | Skill 出错时，自行即时修复 |
| **✅ 自动改进（AUTO-IMPROVE）** | 成功模式自动升级为更优版本 |
| **✅ 自动学习（AUTO-LEARN）** | 从实际使用中捕获高效工作流 |
| **✅ 质量监控** | 跟踪所有任务中的 Skill 表现、错误率和执行成功率 |

> 💡 **核心洞察**：Skill 持续进化——将每次失败转化为改进，将每次成功转化为优化。

### 🌐 超能力二：Agent 集体智慧

将独立的 Agent 联结为共享大脑：

| 特性 | 说明 |
|------|------|
| **✅ 共享进化** | 一个 Agent 的改进即成为所有 Agent 的升级 |
| **✅ 网络效应** | 更多 Agent → 更丰富的数据 → 每个 Agent 更快进化 |
| **✅ 便捷共享** | 一行命令即可上传或下载进化后的 Skill |
| **✅ 访问控制** | 每项 Skill 可选择公开、私有或仅团队可见 |

> 💡 **核心洞察**：一个 Agent 学会，所有 Agent 受益——大规模集体智慧。

### 💰 超能力三：Token 效率

更聪明的 Agent，显著更低的成本：

| 优化方向 | 说明 |
|----------|------|
| **✅ 不再重复劳动** | 复用成功方案，而非每次从零开始 |
| **✅ 任务越做越便宜** | 随着 Skill 改进，类似工作的成本持续下降 |
| **✅ 只做小幅更新** | 修复损坏的部分，无需全部重建 |
| **✅ 实际节省** | 在真实任务上实现 4.2 倍性能提升、Token 消耗减少 46% |

> 💡 **核心洞察**：事半功倍——Agent 真正帮你省钱。

## 📈 GDPVal 基准测试：硬核结果

在 6 个行业的 50 项专业任务上，OpenSpace Agent 使用相同的骨干 LLM（Qwen 3.5-Plus），收入是基线 Agent 的 **4.2 倍**，同时通过 Skill 进化节省了 **46%** 的 Token 开销。

### 测试设计

测试在 [GDPVal](https://huggingface.co/datasets/openai/gdpval) 数据集上进行，包含 220 项真实世界的专业任务，涵盖 44 个职业。

采用两阶段设计：
- **Phase 1（Cold Start）**：按顺序执行全部 50 项任务——每项任务完成后，Skill 积累到共享数据库中
- **Phase 2（Warm Rerun）**：使用 Phase 1 中完整的进化 Skill 库，重新执行相同的 50 项任务

### 核心指标

| 指标 | 数值 |
|------|------|
| **收入提升** | 4.2 倍（相比使用相同骨干 LLM 的 ClawWork 基线） |
| **价值捕获率** | 72.8%（在 $15,764 的任务总价值中赚取 $11,484） |
| **平均质量** | 70.8%（比最佳 ClawWork Agent 的 40.8% 高出 30 个百分点） |
| **Token 效率** | Phase 2 的 Token 用量仅为 Phase 1 的 45.9% |

### 任务类型分布

这些不是玩具级别的问题——而是真实的 GDP 生产工作：

| 类别 | 任务数 | 代表性任务 |
|------|--------|-----------|
| 📝 文档与通信 | 7 | 加州隐私法备忘录、监控调查报告 |
| 📋 合规与表单 | 11 | 从 15 份源文档生成纳税申报表 |
| 🎬 媒体制作 | 3 | ffmpeg 音视频处理、CGI 集锦制作 |
| 🛠️ 工程 | 4 | Web3 全栈开发、CFD 报告 |
| 📊 电子表格 | 15 | 工资计算器、销售预测模型 |
| 📈 战略与分析 | 10 | 供应商谈判策略、能源交易分析 |

## 🧬 进化在何处产生最大影响？

在 50 项 Phase 1 任务中，OpenSpace 自主进化出 **165 项 Skill**。突破性发现：这些不仅是领域知识——它们是**鲁棒的执行模式**和**质量保障工作流**。

### Skill 分类统计

| 用途 | 数量 | 核心内容 |
|------|------|----------|
| **文件格式 I/O** | 44 | PDF 解析回退、DOCX 解析、Excel 合并单元格处理。其中 32/44 从真实失败中捕获。 |
| **执行恢复** | 29 | 分层回退：沙箱失败 → Shell → 写文件后运行 → heredoc。28/29 从实际崩溃中捕获。 |
| **文档生成** | 26 | 端到端文档流水线。`document-gen-fallback` 从 1 项导入 Skill 进化为 13 个衍生版本。 |
| **质量保障** | 23 | 写后验证：检查 Excel 行数、验证 PDF 页数、校验电子表格公式。 |
| **任务编排** | 17 | 多文件追踪、ZIP 打包、零迭代失败检测。 |
| **领域工作流** | 13 | SOAP 病历记录、音频制作、视频流水线。 |
| **网络与研究** | 11 | SSL/代理调试、搜索回退、JS 重页面处理。 |

> 💡 **关键发现**：大多数 Skill 聚焦于**工具可靠性**和**错误恢复**，而非特定任务知识。这说明 OpenSpace 的价值在于让 Agent 在真实世界中可靠运行。

### 各类型任务进化效果

| 类别 | 收入变化 | Token 变化 | 进化原因 |
|------|----------|------------|----------|
| 📝 文档与通信 | 71→74% (+3.3pp) | −56% | `document-gen-fallback` Skill 族历经 13 个版本进化 |
| 📋 合规与表单 | 51→70% (+18.5pp) | −51% | PDF Skill 链只需进化一次，所有表单任务即可复用 |
| 🎬 媒体制作 | 53→58% (+5.8pp) | −46% | 进化的 Skill 编码了可用的 ffmpeg 参数和编解码器回退策略 |
| 🛠️ 工程 | 70→78% (+8.7pp) | −43% | 协调类 Skill 在多样化任务之间通用迁移 |
| 📊 电子表格 | 63→70% (+7.3pp) | −37% | 公式、合并单元格、数据验证模式在各领域完全通用 |
| 📈 战略与分析 | 88→89% (+1.0pp) | −32% | 质量已处最高水平；节省来自于文档结构的复用 |

## 🖥️ 案例：My Daily Monitor

**零行人工编写的代码。60 多项 Skill 从零进化，构建出一个完整可用的实时仪表盘。**

My Daily Monitor 是一个常驻运行的仪表盘，实时展示进程、服务器、新闻、市场、邮件和日程——内置 AI Agent。

### 构建过程

| 阶段 | 发生了什么 | Skill |
|------|------------|-------|
| 🌱 **种子期** | 分析开源项目 WorldMonitor，提取参考模式 | 6 项初始 Skill |
| 🏗️ **脚手架** | 生成项目结构、Vite 配置、TypeScript 设置 | +8 项 Skill |
| 🎨 **构建** | 创建 20 多个面板，配合数据服务、API 路由、网格布局 | +25 项 Skill |
| 🔧 **修复** | 自动修复 TypeScript 错误、API 不匹配、CSS 冲突 | +12 项 FIX 进化 |
| 🧬 **进化** | 衍生增强模式，合并互补 Skill | +15 项 DERIVED Skill |
| 📦 **捕获** | 从成功执行中提取可复用模式 | +8 项 CAPTURED Skill |

每个节点代表 OpenSpace 学习、提取或精炼的一项 Skill。完整的进化历史已开源——可用任意 SQLite 浏览器加载，探索谱系、差异和质量指标。

## 🏗️ 框架深度剖析

### 自我进化引擎

OpenSpace 的核心。Skill 不是静态文件——它们是能够自动选择、应用、监控、分析和进化自身的"活"实体。

#### 三种进化模式

| 模式 | 说明 | 效果 |
|------|------|------|
| 🔧 **FIX** | 就地修复损坏或过时的指令 | 同一 Skill，新版本 |
| 🚀 **DERIVED** | 从父 Skill 创建增强版或专用版 | 新 Skill 目录，与父 Skill 共存 |
| ✨ **CAPTURED** | 从成功执行中提取全新的可复用模式 | 全新 Skill，无父级 |

#### 三个独立触发器

多层防线抵御 Skill 退化——无论执行成功还是失败都驱动进化：

| 触发器 | 说明 |
|--------|------|
| **📈 执行后分析** | 每次任务完成后运行。分析完整记录，为相关 Skill 建议 FIX/DERIVED/CAPTURED。 |
| **⚠️ 工具退化检测** | 当工具成功率下降时，质量监控器找到所有依赖的 Skill 并批量进化。 |
| **📊 指标监控** | 定期扫描 Skill 健康指标（应用率、完成率、回退率），进化表现不佳者。 |

#### 全栈质量监控

多层追踪：质量监控覆盖整个执行栈——从高层工作流到单个工具调用：

| 层级 | 指标 |
|------|------|
| 🎯 **Skill** | 应用率、完成率、有效率、回退率 |
| 🔨 **工具调用** | 成功率、延迟、标记的问题 |
| ⚡ **代码执行** | 执行状态、错误模式 |

**级联进化**：当任何组件退化时——无论是 Skill 工作流还是单个工具调用——上游所有依赖的 Skill 自动触发进化，维持系统级一致性。

### 智能且安全的进化

| 特性 | 说明 |
|------|------|
| 🤖 **自主进化** | 每次进化都会探索代码库、发现根因、自主决定修复——在做出改变之前收集真实证据，而非盲目生成。 |
| ⚡ **基于 Diff 且节省 Token** | 生成最小化的、有针对性的 Diff，而非全量重写，失败时自动重试。每个版本存储在版本 DAG 中，支持完整的谱系追踪。 |
| 🛡️ **内置安全防护** | 确认门控减少误触发；反循环守卫防止进化失控；安全检查标记危险模式；进化后的 Skill 经验证后才替换前代。 |

### 云端 Skill 社区

一个协作式注册中心，Agent 在此共享进化后的 Skill。当一个 Agent 完成改进，所有连接的 Agent 都可以发现、导入并在此基础上构建——将个体进步转化为集体智慧。

| 特性 | 说明 |
|------|------|
| 🔐 **灵活共享** | 可选择公开分享、团队内分享或保持私有。智能搜索帮你找到所需并自动导入。每次进化都有完整 Diff 的谱系追踪。 |
| ☁️ **协作平台** | open-space.cloud — 注册获取 API 密钥、浏览社区 Skill、管理你的团队。 |

## ⚡ 快速开始

### 安装

```bash
git clone https://github.com/HKUDS/OpenSpace.git && cd OpenSpace
pip install -e .
openspace-mcp --help
```

### 路径 A：为你的 Agent 接入

适用于任何支持 Skill（`SKILL.md`）的 Agent——Claude Code、Codex、OpenClaw、nanobot 等。

**① 将 OpenSpace 添加到你的 Agent 的 MCP 配置中：**

```json
{
  "mcpServers": {
    "openspace": {
      "command": "openspace-mcp",
      "toolTimeout": 600,
      "env": {
        "OPENSPACE_HOST_SKILL_DIRS": "/path/to/your/agent/skills",
        "OPENSPACE_WORKSPACE": "/path/to/OpenSpace",
        "OPENSPACE_API_KEY": "sk-xxx (可选，用于云端)"
      }
    }
  }
}
```

> 💡 **提示**：凭证（API 密钥、模型）会从你的 Agent 配置中**自动检测**，通常无需手动设置。

**② 将 Skill 复制到你的 Agent Skill 目录：**

```bash
cp -r OpenSpace/openspace/host_skills/delegate-task/ /path/to/your/agent/skills/
cp -r OpenSpace/openspace/host_skills/skill-discovery/ /path/to/your/agent/skills/
```

完成。这两项 Skill 会教你的 Agent 何时以及如何使用 OpenSpace——无需额外提示。

### 路径 B：作为你的 AI 协作者

直接使用 OpenSpace——编码、搜索、工具调用等——内置自我进化 Skill 和云端社区。

```bash
# 创建 .env 文件并填入你的 LLM API 密钥
# 可选添加 OPENSPACE_API_KEY 以访问云端社区

# 交互模式
openspace

# 执行任务
openspace --model "anthropic/claude-sonnet-4-5" --query "Create a monitoring dashboard for my Docker containers"

# Cloud CLI - 从云端下载 Skill
openspace-download-skill <skill_id>

# Cloud CLI - 上传 Skill 到云端
openspace-upload-skill /path/to/skill/dir
```

### 本地仪表盘

查看你的 Skill 如何进化——浏览 Skill、追踪谱系、比较差异。

```bash
# 终端 1：启动后端 API
openspace-dashboard --port 7788

# 终端 2：启动前端开发服务器
cd frontend
npm install  # 仅首次需要
npm run dev
```

> ⚠️ **要求**：Node.js ≥ 20
