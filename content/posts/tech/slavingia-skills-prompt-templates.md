> 使用方式：本文不是讲原理，而是给你一组可以直接复制、再按自己项目替换的 Prompt 模板。最好的读法不是从头背下来，而是边读边把方括号里的占位符换成你的真实项目。

## 学习目标

读完这篇文章后，你应该能够：

- 快速写出适合自己项目的 Skill 输入，而不是只会输入一句抽象需求。
- 理解不同 Skill 的 Prompt 应该强调哪些信息，避免问得太空或太散。
- 用统一模板降低试错成本，让 Claude Code 的输出更接近真实创业决策。
- 把 10 个 Skill 串成一套自己的执行脚本。

## 先记住一个原则：Skill 不是魔法，输入越具体，输出越值钱

很多人用 Skill 没拿到好结果，不是因为 Skill 不行，而是因为输入太像口号。

比如：

```text
❌ 帮我找一个创业方向
❌ 帮我验证这个想法
❌ 帮我做个 MVP
```

这些说法的问题，不是太短，而是缺少决策所需的最小上下文。Claude Code 不知道：

- 你属于哪个社区。
- 你现在的约束是什么。
- 你是卖服务、卖软件，还是卖内容结果。
- 你最怕犯的错是什么。

更好的 Prompt，不一定更长，但一定更具体。

## 模板总览：每个 Skill 最少要给什么信息

| Skill | 输入里最重要的信息 | 最常见错误 |
| ------ | ------ | ------ |
| /find-community | 你已经属于哪些社区、时间预算、擅长什么 | 直接让它“给我一个点子” |
| /validate-idea | 目标人群、当前替代方案、你观察到的痛点 | 只谈解决方案，不谈问题强度 |
| /mvp | 你想交付的结果、时间限制、不做什么 | 直接让它列功能清单 |
| /processize | 用户输入、你手工步骤、交付输出 | 一上来就问自动化架构 |
| /first-customers | 你现有的人脉、社区连接、可接受销售动作 | 想靠 launch 替代销售 |
| /pricing | 交付成本、用户价值、收入目标 | 不给约束，只问“该定多少钱” |
| /marketing-plan | 已有客户信号、客户常问问题、可投入平台 | 没有成交就开始做内容矩阵 |
| /grow-sustainably | 当前收入、成本、扩张诱惑、不可逆风险 | 把增长只理解成做大 |
| /company-values | 你想坚持什么、不能接受什么、未来合作方式 | 让它写漂亮口号 |
| /minimalist-review | 当前决策、可选方案、担心点、目标 | 问得太泛，只求“给建议” |

## 1. /find-community 模板：别让它替你发明身份

适用时机：你还在找方向，或者点子很多，但不确定该服务谁。

可直接复制的模板：

```text
/find-community
我目前的背景是：[你的职业 / 技能 / 过去经验]。
我已经长期接触的社区有：[社区 A]、[社区 B]、[社区 C]。
我能投入的资源是：[每周时间]、[现金预算]、[现有人脉 / 渠道]。
我更适合做的是：[服务 / 内容 / 工具 / 软件]，不适合做的是：[你不想做的事]。

请不要先给我“大市场创业点子”，而是从我已经真正属于的社区出发，帮我评估：
1. 哪个社区最值得优先服务
2. 这个社区里最持续、最可能付费的问题是什么
3. 我与这个社区的真实连接是否足够强
4. 接下来 7 天我该怎么验证，而不是怎么做产品
```

最关键的替换项不是“想做什么”，而是“你已经属于哪里”。

## 2. /validate-idea 模板：验证的是交易信号，不是礼貌性点赞

适用时机：你已经锁定一个问题，想判断它到底值不值得继续。

可直接复制的模板：

```text
/validate-idea
我正在考虑解决这样一个问题：[用一句话描述问题，而不是描述产品]。
目标人群是：[具体到职业 / 场景 / 平台]。
我观察到他们当前的替代方案是：[现有 workaround]。
我已经看到的痛点信号包括：[重复抱怨 / 加班 / 人工流程 / 付费替代品]。
我的当前约束是：[时间 / 预算 / 技术边界]。

请用 Minimalist Entrepreneur 的方式帮我判断：
1. 这个问题是不是足够痛
2. 哪些信号算强验证，哪些只是礼貌性反馈
3. 我现在最该做的 5 个验证动作是什么
4. 什么情况下我应该继续，什么情况下我应该暂停或转向
```

如果你想让输出更实用，再补一句：

```text
请优先给我“这周就能执行”的动作，不要先给我长期商业规划。
```

## 3. /mvp 模板：让它帮你收缩结果，不是扩张功能

适用时机：你已经知道问题存在，但总忍不住往产品功能上加东西。

可直接复制的模板：

```text
/mvp
我想解决的问题是：[问题]。
我想服务的人是：[具体人群]。
我希望给他们带来的结果是：[结果，不是功能]。
我目前能投入的开发 / 交付窗口只有：[例如 1 个周末 / 2 周]。
我明确不想做的事情有：[列出 3 到 5 项]。

请帮我定义一个真正的 MVP：
1. 它必须只解决一个核心问题
2. 它必须能在我的时间限制内推出
3. 它最好从第一天就能收费
4. 它必须便于快速拿反馈

请输出：
- 一句话 MVP 定义
- 必做项
- 明确不做项
- 最快交付方式
- 本周的执行顺序
```

这里最容易被忽视的一句是“明确不想做的事情”。没有这一句，Skill 很容易回到功能扩张。

## 4. /processize 模板：先把手工流程跑通，别急着系统化

适用时机：你已经知道交付什么结果，但还没把手工交付流程写清楚。

可直接复制的模板：

```text
/processize
我想交付的核心结果是：[一句话写清楚客户最终拿到什么]。
目标客户会提供给我的输入是：[文件 / 链接 / 文本 / 表单]。
我目前预想的手工步骤是：[你已经想到的 3 到 7 步]。
我最终交付的输出是：[文档 / 报告 / 脚本 / 文件]。
我希望先服务的对象是：[3 个具体人或一类具体人]。

请帮我把这件事 processize：
1. 把手工流程拆成明确步骤
2. 指出哪一步最容易出错
3. 指出哪一步最耗时
4. 帮我写成一张别人也能照着执行的 magic piece of paper
5. 判断哪些部分暂时不要自动化
```

这类 Prompt 的目标不是“更聪明”，而是“更可执行”。

## 5. /first-customers 模板：先卖给最接近你的人

适用时机：你已经能交付结果，准备开始拿首批客户。

可直接复制的模板：

```text
/first-customers
我现在提供的结果是：[一句话服务定义]。
我的目标客户是：[具体人群]。
我当前最接近他们的连接包括：[朋友 / 前同事 / 社群 / 私域 / 公众号 / 线下圈子]。
我目前最担心的销售问题是：[不敢开口 / 表达不清 / 不知道找谁]。
我希望在 [时间范围] 内拿到 [客户数量] 个付费客户。

请帮我制定首批客户策略：
1. 按朋友 / 社区 / 冷启动分层排序
2. 每一层我应该优先联系谁
3. 给我一版不油腻的外联话术
4. 给我一版跟进节奏
5. 告诉我哪些行为看似忙碌，其实不能带来成交
```

如果你已经知道几个具体人名，可以继续补：

```text
下面是我准备先联系的 10 个人：[名单或特征]。请帮我按成交可能性排序。
```

## 6. /pricing 模板：价格不是面子问题，是信号问题

适用时机：你已经能交付结果，但不确定怎么收费。

可直接复制的模板：

```text
/pricing
我当前提供的结果是：[一句话服务或产品描述]。
目标客户是：[具体人群]。
我的主要成本包括：[时间 / 工具费 / 交付成本 / 第三方费用]。
客户可能获得的价值包括：[省时间 / 省钱 / 提升收入 / 降低错误率]。
我希望达到的短期目标是：[例如 月入 5000 元 / 验证是否有人付费]。

请帮我设计初始定价：
1. 给出一个适合当前阶段的价格结构
2. 说明这个价格为什么适合验证，而不是只适合赚钱
3. 帮我设计何时涨价的判断标准
4. 告诉我什么情况下应该继续低价，什么情况下应该提价
```

如果你不想被抽象回答带偏，可以补一句：

```text
请优先从“我现在如何更快获得付费信号”来设计价格，而不是从成熟 SaaS 模式出发。
```

## 7. /marketing-plan 模板：只有当你已经有客户语言，内容才不会空

适用时机：已经出现稳定成交、复购或清晰客户语言时。

可直接复制的模板：

```text
/marketing-plan
我当前已经有的验证信号是：[付费客户数 / 复购 / 访谈 / 社区反馈]。
我的客户最常问的问题是：[列 3 到 5 个]。
我擅长表达的内容形式是：[短文 / 长文 / 视频 / 推文 / 播客]。
我实际能长期维护的平台只有：[1 到 2 个平台]。

请帮我制定一个极简但可持续的营销计划：
1. 先做什么内容，不做什么内容
2. 内容如何直接连接到销售而不是只做曝光
3. 未来 4 周我该输出哪些主题
4. 哪些内容最适合 educate、inspire、entertain
```

如果你还没有客户语言，请先不要用这个模板，先回去做 /first-customers。

## 8. /grow-sustainably 模板：把“看起来像增长”的动作筛掉

适用时机：你开始面对花钱、扩张、招聘、投放等经营决策。

可直接复制的模板：

```text
/grow-sustainably
我当前的项目状态是：[收入水平 / 客户数量 / 交付方式 / 成本结构]。
我现在在考虑的扩张动作是：[招人 / 投广告 / 做后台 / 租办公室 / 融资]。
我担心的问题是：[现金流 / 交付质量 / 时间不足 / 增长焦虑]。

请帮我用 Minimalist Entrepreneur 的方式复核：
1. 这一步是否真的有必要
2. 有没有更便宜、更可逆的替代动作
3. 它对盈利能力和控制权的影响是什么
4. 我现在最该保住的是什么，不该急着追的是什么
```

这类 Prompt 的关键不在信息量，而在你是否诚实地写出真实收入与成本状态。

## 9. /company-values 模板：不要让它帮你生成漂亮废话

适用时机：你已经开始考虑长期合作、业务边界和组织方式。

可直接复制的模板：

```text
/company-values
我正在做的业务是：[一句话业务定义]。
我希望这门生意长期保持的特点是：[例如 小团队 / 高利润 / 远程 / 深度服务]。
我明确不想成为的样子是：[例如 全靠投流 / 低价内卷 / 客户什么都接]。
我过去最认同的做事原则是：[列 3 条]。
我最不能接受的合作行为是：[列 3 条]。

请帮我提炼 3 到 5 条真正能指导经营决策的价值观：
1. 每条都要说明它具体意味着什么
2. 每条都要说明它不意味着什么
3. 每条都要能落到招聘、合作或产品决策上
```

如果输出里全是“客户第一、追求卓越、开放协作”这种套话，说明你给的输入还不够具体。

## 10. /minimalist-review 模板：任何让你兴奋的新想法，都值得先过这一关

适用时机：出现一个看上去很诱人的新功能、新渠道、新方向或新合作。

可直接复制的模板：

```text
/minimalist-review
我现在要做一个决策：[一句话写清楚决策内容]。
我考虑的原因是：[为什么现在想做]。
我担心的地方是：[成本 / 偏航 / 时间 / 复杂度 / 风险]。
我当前真正的目标是：[例如 先拿 10 个付费客户 / 先稳定复购 / 保持盈利]。

请用 Minimalist Entrepreneur 的原则帮我复核：
1. 这件事是否真的服务客户或社区
2. 有没有更简单、更便宜、更可逆的版本
3. 它会让我更接近核心目标，还是只是让我看起来更像一家“大公司”
4. 给我一个明确结论：现在做 / 以后再做 / 不做
```

## 一个组合模板：如果你不知道今天该调用哪个 Skill

有时候你不是不会写 Prompt，而是根本不知道当前处在哪一步。这个时候可以先用一个总诊断模板：

```text
我在做一个项目：[一句话项目描述]。
我当前的状态是：[有想法 / 已访谈 / 已成交 / 已复购 / 已开始内容增长]。
我当前最大的卡点是：[不知道服务谁 / 不确定值不值得做 / 产品范围失控 / 不敢收费 / 找不到客户 / 想扩张但担心失控]。
我当前最真实的约束是：[时间 / 现金 / 人脉 / 技术边界]。

请告诉我，在 Minimalist Entrepreneur Skills 这 10 个 Skill 里，我现在最应该先用哪一个，并说明原因。然后请给我一版适合当前阶段的输入模板。
```

## 最后提醒：不要把模板当护身符

这些模板能显著提高你拿到有效输出的概率，但它们解决不了一个根本问题：如果你输入的不是事实，而是愿望，输出再漂亮也不会变成商业验证。

所以真正高价值的用法不是“把模板写得更像模板”，而是把里面每个占位符都替换成你已经观察到、已经经历过、已经承担过的真实约束。
