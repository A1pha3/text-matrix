## 二、为什么需要"生产级"SKILL

### 2.1 常见问题

很多人在开发 SKILL 时会遇到以下问题：

| 问题 | 表现 | 根因 |
|------|------|------|
| **运行不稳定** | 同样的输入，有时成功有时失败 | 流程没有固定 |
| **输出质量参差** | 每次输出的格式、风格不一致 | 缺乏模板约束 |
| **难以调试** | 出了问题不知道哪里错 | 缺少错误处理 |
| **无法批量生产** | 一个一个手动做，效率低 | 缺乏自动化思维 |

### 2.2 生产级的标准

一个"生产级"的 SKILL 应该满足：

1. **确定性**：同样的输入，一定产生同等的输出
2. **稳定性**：99% 以上的运行成功率
3. **可复用性**：换一个主题/内容，依然能正常工作
4. **可维护性**：出了问题容易定位和修复

## 三、核心方法论：模板化

### 3.1 什么是模板化

**模板化**就是把 SKILL 的工作流程用代码/逻辑固定下来，而不是依赖 AI 的"随意发挥"。

```
❌ 错误思路：让 AI 自己决定怎么做
   → AI 的输出不可预测，SKILL 运行不稳定

✅ 正确思路：把流程固定，让 AI 只负责执行特定步骤
   → SKILL 运行稳定，输出可预期
```

### 3.2 模板化的三个层次

| 层次 | 说明 | 示例 |
|------|------|------|
| **流程模板** | 固定工作流程的顺序 | 输入 → 处理 → 输出 |
| **格式模板** | 固定输出格式和风格 | HTML 结构、CSS 样式 |
| **内容模板** | 固定内容生成规则 | 变量占位、循环结构 |

### 3.3 实践案例：产品爆炸图网页

视频中演示的案例是**批量生成 Apple 风格的产品爆炸图网页**。

这种网页的特点：
- 多角度产品展示
- 精致的动画效果
- 统一的视觉风格

如果不用模板化，每次生成都会不一样。有了模板化：
1. 定义好 HTML 结构模板
2. 定义好 CSS 样式模板
3. AI 只需要填充产品图片和参数

## 四、核心方法论：先拆分后融合

### 4.1 为什么需要拆分

复杂的 SKILL 如果一气呵成，容易出问题：

- 步骤太多，调试困难
- 一个环节出错，整个 SKILL 失败
- 难以精准控制每一步的质量

### 4.2 拆分策略

把复杂的 SKILL 拆分成多个简单的子任务：

```
复杂任务：生成完整的产品爆炸图网页
    ↓ 拆分成：
子任务 1：生成 HTML 结构
子任务 2：生成 CSS 样式
子任务 3：生成 JavaScript 交互
子任务 4：生成首尾帧视频
    ↓ 最后融合：
最终输出：完整的产品爆炸图网页
```

### 4.3 融合技巧

拆分后的融合需要注意：

1. **接口统一**：每个子任务的输出格式要一致
2. **顺序控制**：明确哪些可以并行，哪些必须串行
3. **质量检查**：每一步完成后检查输出是否符合预期

视频中提到，作者把产品爆炸图 SKILL 拆分成了**两步**：
1. 第一步：生成静态网页（HTML + CSS + JS）
2. 第二步：生成首尾帧视频
3. 最后：合并到一起

### 6.2 SKILL 开发流程

**第一步：明确目标**
- 确定 SKILL 要解决什么问题
- 定义输入和输出格式
- 设定成功标准

**第二步：流程设计**
- 把任务拆分成多个步骤
- 确定每个步骤的输入输出
- 设计融合策略

**第三步：模板编写**
- 编写固定的流程模板
- 定义输出格式模板
- 预留变量占位符

**第四步：分步实现**
- 先实现第一步，测试稳定
- 再实现第二步，测试稳定
- 最后实现融合逻辑

**第五步：压力测试**
- 用多种输入测试 SKILL 稳定性
- 修复发现的问题
- 优化性能和输出质量

## 八、常见问题

### Q: 为什么我的 SKILL 运行总是不稳定？

A: 很可能是没有做好模板化。检查以下几点：
- 流程是否固定？有没有让 AI 随意决定的部分？
- 输出格式是否定义清晰？
- 有没有做好错误处理？

### Q: 拆分的粒度应该多细？

A: 经验法则是：**每一步应该可以独立测试成功**。如果一步太复杂，就继续拆分。

### Q: MiniMax-M2.7 和 M2.5 有什么区别？

A: M2.7 专门为 OpenClaw 设计，Agent 能力进行了优化。如果你的 SKILL 需要多轮对话、复杂推理、工具调用，M2.7 表现会更好。

### Q: 海螺 2.0 首尾帧适合什么场景？

A: 适合需要**视频与图片风格一致**的场景，比如：
- 产品展示视频
- 品牌动画
- 教程视频片头
