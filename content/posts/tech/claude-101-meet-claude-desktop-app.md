## Claude 桌面应用简介

Claude 桌面应用是 Anthropic 官方提供的桌面客户端，让你在电脑上更方便地使用 Claude。相比网页版，桌面应用有一些独特优势：

| 功能 | 网页版 | 桌面应用 |
|------|--------|----------|
| 随时呼出 | 需要打开浏览器 | 全局快捷键 |
| 文件交互 | 上传文件 | 直接拖拽文件 |
| 代码执行 | 只能看代码 | 内置代码执行环境 |
| 快捷操作 | 无 | 自定义快捷方式 |
| 系统集成 | 无 | 更深入的操作系统集成 |

---

## 三种模式概述

Claude 桌面应用提供三种不同的工作模式：

| 模式 | 适用场景 | 核心能力 |
|------|----------|----------|
| **Chat** | 日常对话、问答、写作 | 快速问答、创意头脑风暴 |
| **Cowork** | 团队协作、项目工作 | 与 Claude 共同完成项目 |
| **Code** | 编程任务、代码审查 | 专业级代码助手 |

---

## Chat 模式

### 什么时候用 Chat？

Chat 模式适合**日常快速任务**，比如：

- 回答一个问题
- 写一封邮件
- 翻译一段文字
- 头脑风暴创意点子
- 解释一个概念

### Chat 模式的特点

- **对话式**：像跟人聊天一样简单直接
- **轻量级**：不需要设置，直接开始
- **上下文有限**：适合短时任务

### 使用示例

```
用户：帮我翻译这段话成英文，要保持原文的语气

Claude：[完成翻译]
```

## Cowork 模式

### 什么时候用 Cowork？

Cowork 模式适合**需要 Claude 参与整个项目**的场景：

- 写一本书或长篇文章
- 开发一个完整的产品
- 制定商业计划
- 规划一次旅行
- 准备一场演讲

### Cowork 模式的特点

| 特性 | 说明 |
|------|------|
| 持久上下文 | Claude 记住整个项目的所有细节 |
| 版本管理 | 可以追踪项目的不同版本 |
| 多人协作 | 可以邀请团队成员一起参与 |
| 项目记忆 | Claude 记得你之前在项目中做的决定 |

### 核心概念：Projects

在 Cowork 模式中，你的所有工作都在 **Projects** 中进行：

```
Project（项目）
├── 对话历史
├── 共享上下文（你告诉 Claude 的背景信息）
├── 生成的文件
└── 重要决策记录
```

### 使用示例

```
你正在写一本书：

第一步：创建项目
- 项目名称：我的第一本书
- 主题：AI 时代的学习方法
- 目标读者：大学生和职场新人

第二步：开始对话
"帮我规划这本书的章节结构"

第三步：逐章写作
"现在写第一章的第一节……"

第四步：迭代修改
"第三章的第二部分有点太学术了，改得更通俗一点"
```

## Code 模式

### 什么时候用 Code？

Code 模式专门为**编程任务**设计，适合：

- 写新代码
- 调试和修复 bug
- 代码审查
- 重构代码
- 学习新语言/框架

### Code 模式的特点

| 特性 | 说明 |
|------|------|
| 代码执行 | 内置 Python/JavaScript 等运行环境 |
| 文件操作 | 直接读取、编辑你的代码文件 |
| Git 集成 | 可以查看 git 历史、创建提交 |
| 终端访问 | 可以在应用内运行终端命令 |

### 使用示例

```
你有一个 bug：

1. 把代码文件拖入 Claude Code
2. "这段代码报错了，请帮我分析问题"
3. Claude 分析代码，找出问题
4. "请修复这个 bug"
5. Claude 给出修复方案并解释原因
6. "应用这个修复"
7. Claude 直接修改文件
```

### Code 模式支持的语言

Claude Code 原生支持：

- Python
- JavaScript / TypeScript
- Go
- Rust
- Java
- C / C++
- 以及几乎所有主流编程语言

## 三种模式对比

### 核心差异

| 维度 | Chat | Cowork | Code |
|------|------|--------|------|
| **交互方式** | 短对话 | 长期项目 | 任务导向 |
| **上下文** | 单次会话 | 项目级持久 | 文件级 |
| **文件操作** | 有限 | 支持 | 完整支持 |
| **代码执行** | ❌ | ❌ | ✅ |
| **版本管理** | ❌ | ✅ | ✅ |
| **最适合** | 快速问答 | 长期项目 | 编程任务 |

### 什么时候选哪个？

```
任务类型判断流程：

1. 这个任务是编程相关的吗？
   → 是 → 用 Code 模式
   → 否 → 继续判断

2. 这个任务需要持续几天或几周吗？
   → 是 → 用 Cowork 模式（创建项目）
   → 否 → 继续判断

3. 这个任务简单吗？（回答一个问题、写一段文字）
   → 是 → 用 Chat 模式
   → 否 → 用 Cowork 模式
```

## 快速开始指南

### 第一步：下载安装

1. 访问 [claude.ai](https://claude.ai) 下载页面
2. 选择对应操作系统的版本（macOS / Windows）
3. 安装并登录你的账号

### 第二步：熟悉界面

```
┌─────────────────────────────────────────────────┐
│  [Chat]  [Cowork]  [Code]     [设置]  [用户]   │  ← 模式切换 + 导航
├─────────────────────────────────────────────────┤
│                                                 │
│  ┌─────────────────────────────────────────┐  │
│  │                                         │  │
│  │           对话区域                       │  │
│  │                                         │  │
│  └─────────────────────────────────────────┘  │
│                                                 │
│  ┌─────────────────────────────────────────┐  │
│  │ 输入框...                    [发送] 📎  │  │  ← 输入区 + 附件
│  └─────────────────────────────────────────┘  │
│                                                 │
└─────────────────────────────────────────────────┘
```

### 第三步：选择模式并开始

- Windows：`Ctrl + 1/2/3` 切换模式
- macOS：`Cmd + 1/2/3` 切换模式

### 全局快捷键

| 功能 | Windows | macOS |
|------|---------|-------|
| 呼出 Claude | `Ctrl + Shift + C` | `Cmd + Shift + C` |
| 新对话 | `Ctrl + N` | `Cmd + N` |
| 搜索对话 | `Ctrl + K` | `Cmd + K` |

## 实用技巧

### 技巧一：在 Chat 中快速切换到 Cowork

如果你在 Chat 模式中开始了一个项目，可以随时：
1. 点击右上角的「移动到项目」
2. 选择「创建新项目」
3. 对话历史会被保留

### 技巧二：在 Code 中使用 Claude 的代码解释功能

把一段代码拖入 Claude Code，然后问：
- "这段代码是做什么的？"
- "有没有潜在的性能问题？"
- "这段代码可以怎么改进？"

### 技巧三：利用 Cowork 的项目记忆

在项目设置中，你可以预设：
- 项目背景
- 你的偏好（比如"总是用中文回答"）
- 常用文件路径
- 团队成员信息

这样 Claude 在每次对话中都知道这些背景，不用你重复解释。

1. 写一个简单的 Python 脚本（有意的 bug）

3. 让 Claude 修复它
