## 学习目标

学完本节后，你将能够：

- 理解什么是 Project（项目）以及为什么需要它
- 掌握如何创建、管理和使用 Project 进行长期工作
- 深入理解 Artifacts（产物）是什么，以及如何在日常工作中高效使用
- 学会利用 Skills（技能）扩展 Claude 的能力边界
- 建立一套完整的「人机协作」工作流程，让 AI 真正成为你的智能助手

## 为什么你需要「组织」这件事？

在正式讲解工具之前，让我们先思考一个问题：**为什么用 AI 助手需要「组织」？**

想象一下，你请了一个能力超强的助理。但每次你去找他，他都不记得：
- 你上次跟他讨论的项目是什么
- 你们之前达成了什么结论
- 你给他看过的文件在哪里
- 你的偏好和工作习惯是什么

结果每次你都要从头解释一遍，效率极其低下。

**普通的 AI 对话就是这样**——每次新建对话，AI 就「失忆」了，它不记得你上次说了什么，更不知道你的项目背景。

Claude 的 Projects、Artifacts 和 Skills 这三个功能，就是为了解决这个根本性问题而设计的。它们让 Claude 从一个「一次性的问答机器」，变成一个「懂你、记得你、持续帮助你的智能助手」。

## 一、Projects（项目）：Claude 的记忆宫殿

### 什么是 Project？

**Project** 是 Claude 提供的长期上下文管理机制。你可以把它理解为：

> **一个带记忆的对话空间** —— Claude 在这个空间里记得所有对话历史、所有你告诉它的背景信息、所有你们一起产生的成果。

### 为什么需要 Project？

让我们用一个真实的场景来理解：

**没有 Project 的日常**：

```
你：帮我写一个产品需求文档 PRD
Claude：好的，请告诉我产品名称、目标用户、核心功能...

你：是一个在线教育平台，面向 22-35 岁职场人士...
Claude：明白了，还有更多细节吗？
（你花了 10 分钟描述背景）

Claude：好的，这是 PRD 的框架...
（Claude 开始写）

第二天，你新建对话：
你：继续完善昨天那个 PRD
Claude：抱歉，我不知道你说的是什么 PRD，也没有之前的上下文
```

**有 Project 的日常**：

```
你（创建了一个 Project 叫「在线教育 PRD」）：
帮我写一个产品需求文档 PRD，是一个在线教育平台，面向 22-35 岁职场人士...

Claude：好的，这是 PRD 的框架...
（Claude 开始写，并记住了所有背景）

第二天，你继续同一个 Project：
你：继续完善昨天那个 PRD，增加第三章的市场分析

Claude：好的，我记得这是一个面向 22-35 岁职场人士的在线教育平台，
第二章我们写完了核心功能，现在继续第三章市场分析...
```

### Project 的核心价值

| 价值维度 | 具体说明 |
|----------|----------|
| **上下文连续性** | Claude 记得整个项目的所有历史对话和决策 |
| **背景知识沉淀** | 你告诉 Claude 的项目背景、品牌调性、目标读者，都会被记住 |
| **成果统一管理** | 所有生成的文件、文档、代码，都在一个项目里 |
| **团队协作** | 可以邀请团队成员加入项目，大家共享上下文 |

### 如何创建一个 Project？

### 方法一：从零开始

```
1. 打开 Claude 桌面应用
2. 点击「新建 Project」
3. 输入项目名称和描述
4. （可选）上传相关文件作为初始上下文
5. 开始对话
```

### 方法二：从现有对话迁移

```
1. 在 Chat 模式中已经有一段有价值的对话
2. 点击右上角「移动到 Project」
3. 选择「创建新项目」或「加入已有项目」
4. 对话历史会被完整保留
```

### Project 的最佳实践

#### 实践一：按「工作域」划分项目

不要把所有事情都塞进一个 Project。建议这样组织：

| Project 名称 | 包含内容 |
|-------------|----------|
| 📚 我的读书计划 | 所有读书笔记、书评、读后感 |
| 💼 产品经理工作台 | PRD、竞品分析、用户研究 |
| 🛠️ 独立开发项目 | 代码、架构文档、技术方案 |
| 📝 个人博客 | 所有博客文章、选题灵感、草稿 |

#### 实践二：项目设置中预设上下文

在项目设置中，你可以预先填写：

```
项目名称：在线教育平台 PRD
项目描述：一个面向职场人士的在线学习平台

背景信息：
- 目标用户：22-35 岁职场人士，碎片化学习
- 核心价值主张：随时随地学习，不浪费时间

团队成员：
- 产品负责人：张三
- 技术负责人：李四

常用文件：
- 用户调研报告：/docs/research.pdf
- 竞品分析：/docs/competitor.md
```

Claude 会在每次对话中自动加载这些背景，不用你重复输入。

#### 实践三：定期「整理」项目

Project 不是一次性工具，需要定期维护：

- **每周回顾**：花 5 分钟回顾本周在项目中的进展
- **归档完成的任务**：把已经完成的工作归档，保持项目整洁
- **更新上下文**：如果项目方向调整，及时更新项目背景

## 二、Artifacts（产物）：让 AI 生成的内容「活」起来

### 什么是 Artifacts？

**Artifacts** 是 Claude 在对话过程中生成的「可持久化内容」。它可以是：

- 一段代码
- 一篇文章
- 一个网页原型
- 一个图表
- 任何你可以「复制走、用起来」的东西

### 为什么需要 Artifacts？

普通的 AI 对话，生成的内容「活在」对话里，你想要就得复制粘贴走。如果内容需要更新，你得：
1. 记住是在哪次对话
2. 找到那条消息
3. 复制内容
4. 粘贴到你的文档/代码里

**Artifacts 改变了这个流程**——生成的内容可以直接「独立存在」，你可以：
- 一键复制
- 实时预览效果
- 持续迭代更新
- 直接分享给他人

### Artifacts 的典型使用场景

#### 场景一：代码片段

```
你：帮我写一个 Python 函数，实现快速排序

Claude（生成 Artifact）：
def quicksort(arr):
    if len(arr) <= 1:
        return arr
    pivot = arr[len(arr) // 2]
    left = [x for x in arr if x < pivot]
    middle = [x for x in arr if x == pivot]
    right = [x for x in arr if x > pivot]
    return quicksort(left) + middle + quicksort(right)
```

这个代码块会以 Artifact 形式呈现，你可以：
- 一键复制
- 看到运行效果
- 让他帮你解释每一行

#### 场景二：网页/UI 原型

```
你：用 HTML 和 CSS 写一个登录页面的原型

Claude（生成 Artifact）：
一个完整的、可视化的 HTML 页面，你可以直接在浏览器中打开预览
```

#### 场景三：文档和文章

```
你：帮我写一篇产品介绍文章

Claude（生成 Artifact）：
一篇格式规范的 Markdown 文章，你可以复制到任何博客平台
```

### Artifacts vs 普通回复：核心区别

| 维度 | 普通回复 | Artifacts |
|------|----------|-----------|
| **持久性** | 随对话消失 | 可以独立存在、随时访问 |
| **可编辑性** | 只能复制走 | 可以直接在 Artifact 上编辑 |
| **可分享性** | 截图/复制 | 生成链接一键分享 |
| **适用内容** | 简单回答 | 代码、文档、设计稿等结构化内容 |

### 如何高效使用 Artifacts？

#### 技巧一：让 Claude「迭代」Artifact

```
第一轮：
你：帮我写一个登录页面的 HTML

Claude（生成了 Artifact v1）

第二轮：
你：登录按钮改成圆角的，再加一个"记住我"的复选框

Claude（更新 Artifact v2，你看到实时变化）

第三轮：
你：配色改成深色主题

Claude（更新 Artifact v3）
```

#### 技巧二：指定 Artifact 的「角色」

```
你：写一个 Python 类，用来管理用户数据。用「生产级」的标准来写，
包含类型注解、文档字符串、异常处理。
```

这样说，Claude 会按照更高标准来生成代码。

#### 技巧三：用 Artifact 做「版本控制」

如果 Artifact 更新了很多版本，想要回溯：

```
你：把之前的代码回滚到第一个版本，然后告诉我那个版本的优缺点
```

## 三、Skills（技能）：让 Claude 做它本来「不会」的事

### 什么是 Skills？

**Skills** 是 Claude 的扩展能力模块。你可以理解为：

> **给 Claude 装的「插件」** —— 每个 Skill 让 Claude 能做某个特定领域的专业任务。

Claude 官方提供了一系列内置 Skills，比如：
- 浏览器搜索（实时获取网络信息）
- 网页抓取（读取网页内容）
- 文件操作（读写本地文件）
- 代码执行（运行代码并返回结果）

### 为什么需要 Skills？

#### 没有 Skills 的局限

```
你：帮我查一下今天比特币的价格

Claude：抱歉，我无法访问实时数据...
```

#### 有 Skills 的能力

```
你：帮我查一下今天比特币的价格（开启了 Browser Skill）

Claude：好的，让我查一下...

【Claude 调用 Browser Skill 访问 CoinMarketCap】
当前比特币价格：$67,234.56（截至 2024年3月25日 10:00）
24 小时涨跌：+2.34%
```

### Skills 的工作原理

Skills 本质上是给 Claude 提供了一种「调用工具」的能力：

```
用户请求 → Claude 理解意图 → 选择合适的 Skill → Skill 执行 → 结果返回 Claude → Claude 整合回答
```

### 常用的内置 Skills

| Skill | 能力 | 使用场景 |
|-------|------|----------|
| **Browser** | 搜索和读取网页 | 查实时信息、读文章 |
| **File Access** | 读写本地文件 | 读取文档、分析代码库 |
| **Code Execution** | 执行代码 | 运行脚本、验证代码 |
| **Terminal** | 使用终端命令 | 操作服务器、执行 CLI |
| **Image Generation** | 生成图片 | 配图、图示 |

### 如何使用 Skills？

#### 方法一：在对话中启用

```
你：帮我分析一下这篇网页的内容（Browser Skill 会在需要时自动启用）
```

#### 方法二：Project 中默认启用

在 Project 设置中，可以预设开启哪些 Skills，这样 Claude 在这个项目中就会默认可以使用这些能力。

### Skills 的进阶用法：组合使用

Skills 可以组合使用，实现复杂任务：

```
你：帮我分析一下这个 GitHub 仓库的代码结构，然后运行一下测试用例，
最后生成一份总结报告

【Claude 执行流程】
1. File Access Skill → 读取仓库结构
2. Code Execution Skill → 运行测试
3. Claude 整合分析 → 生成报告
```

## 四、三者的协同价值：完整的「人机协作」工作流

### 为什么它们组合在一起更强大？

Projects、Artifacts 和 Skills 不是孤立的功能，它们组合起来，形成了一套完整的人机协作体系：

```
Projects（记忆）
    ↓  提供上下文背景
Artifacts（产出）
    ↓  承载生成内容
Skills（工具）
    ↑  提供执行能力
```

### 实战场景：用 Claude 完成一本书的写作

**场景**：你要写一本关于「AI 时代个人成长」的书

#### 第一步：创建项目（Projects）

```
创建 Project：「AI 成长手册」

项目背景：
- 书名：AI 时代的个人成长指南
- 目标读者：想利用 AI 提升效率的职场人士
- 调性：专业但不学术，通俗但不浅薄
- 篇幅：约 8 万字，15 章
```

#### 第二步：制定大纲（Artifacts + Skills）

```
你：帮我制定这本书的大纲结构

Claude（生成 Artifact v1）：
├── 第一部分：认知篇
│   ├── 第一章：AI 是什么，不是什么
│   ├── 第二章：AI 能做什么，不能做什么
│   └── 第三章：为什么你现在就必须学会用 AI
├── 第二部分：工具篇
│   ├── 第四章：Claude 入门
...
```

#### 第三步：逐章写作（Artifacts 迭代）

```
你（继续 Project 对话）：继续写第一章的第一节

Claude（生成 Artifact）：...

你：第三节的案例太老套了，换成最近的案例

Claude（更新 Artifact）：...
```

#### 第四步：使用 Skills 扩展能力

```
你：帮我搜索一下最近关于 AI和个人成长的研究论文

【Browser Skill 自动启用】
Claude：[搜索结果列表]

你：把这些论文的核心观点整合到第三章里

Claude：[更新章节内容]...
```

#### 第五步：最终整理（Artifacts 导出）

```
你：把所有章节整理成一份完整的目录和内容清单

Claude（生成 Artifact）：
- 书名：AI 时代的个人成长指南
- 总字数：约 8 万字
- 章节数：15 章
- 状态：大纲 ✅ / 第一章 ✅ / 第二章 进行中...
```

## 五、常见问题与最佳实践

### Q1：Project 太多了怎么办？

**建议**：按「工作域」而不是「任务」来组织 Project。

- ❌ 一个 Project 叫「写 PRD」
- ✅ 一个 Project 叫「产品经理工作台」，包含所有产品相关工作

### Q2：Artifact 太多，找不到想要的怎么办？

**建议**：
- 给 Artifact 起描述性的名字
- 在 Project 中定期整理，删除不再需要的 Artifact
- 使用 Project 的「搜索」功能

### Q3：Skills 会不会泄露我的隐私？

**说明**：
- Skills 的调用需要你授权
- 敏感操作（如文件访问）会明确提示
- 你可以在 Project 设置中选择禁用某些 Skills

### Q4：应该把所有东西都放在 Claude 里吗？

**建议**：不要把 Claude 当成「存储服务」。

适合放进去的：
- 对话上下文和背景信息
- 正在进行的项目
- AI 帮你生成的草稿和内容

不适合放进去的：
- 机密文件（密码、密钥、私人数据）
- 大量历史文档（用文件管理工具管理）
- 不需要 AI 理解的纯存档
