# Open Higgsfield AI：2.7K Stars·开源NotebookLM·文档转播客·AI双主持·引用溯源

## 一，项目概述

### 1.1 Open Higgsfield AI 是什么

**Open Higgsfield AI** 是 Google **NotebookLM** 的**开源复现**，将任何研究文档转化为**AI 双主持播客**。

> "Open-source NotebookLM. Transform any research document into a podcast with AI-generated hosts."

### 1.2 核心数据

| 指标 | 数值 |
|------|------|
| Stars | **2.7k** ⭐ |
| Forks | 193 |
| 贡献者 | 6 |
| 最新版本 | **v1.0.0** (2026-03-29) |
| 许可证 | MIT |
| 语言 | Python 100% |

### 1.3 核心定位

| 维度 | 说明 |
|------|------|
| 📄 **文档处理** | PDF、网站、文本 |
| 🎙️ **播客生成** | AI 双主持 |
| 🔗 **引用溯源** | 实时来源标注 |
| 💬 **交互问答** | 深入理解内容 |

### 1.4 核心特性

| 特性 | 说明 |
|------|------|
| ✅ **文档上传** | PDF、网页、文本 |
| ✅ **AI 播客** | 双主持自动生成 |

## 四、安装部署

### 4.1 环境要求

| 要求 | 版本 |
|------|------|
| Node.js | 18+ |
| npm | 9+ |
| OpenAI API Key | 必需 |
| Convex 账号 | 必需 |
| Clerk 账号 | 必需 |

### 4.2 快速安装

```bash
# 克隆仓库
git clone https://github.com/Anil-matcha/Open-Higgsfield-AI.git
cd Open-Higgsfield-AI

# 安装依赖
npm install

# 配置环境变量
cp .env.example .env.local
```

### 4.3 环境变量配置

```bash
# .env.local

# OpenAI（必需）
OPENAI_API_KEY=sk-...

# Convex（必需）
CONVEX_DEPLOYMENT_URL=https://xxx.convex.cloud
CONVEX_DEPLOYMENT_KEY=xxx
NEXT_PUBLIC_CONVEX_URL=https://xxx.convex.cloud

# Clerk 认证（必需）
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=pk_test_xxx
CLERK_SECRET_KEY=sk_test_xxx
```

### 4.4 启动开发服务器

```bash
# 启动 Convex 后端
npx convex dev

# 在另一个终端启动 Next.js 前端
npm run dev
```

### 4.5 生产环境部署

```bash
# 构建生产版本
npm run build

# 启动生产服务器
npm start
```

## 五、使用指南

### 5.1 首次使用

```
1. 访问应用
   └─► 打开 http://localhost:3000

2. 注册/登录
   └─► 通过 Clerk 认证

3. 创建项目
   └─► 点击 "New Project"

4. 上传文档
   └─► 支持 PDF/URL/文本

5. 生成播客
   └─► 点击 "Generate Podcast"
```

### 5.2 上传文档

| 方式 | 操作 |
|------|------|
| **拖拽** | 直接拖拽文件到上传区 |
| **点击** | 点击选择文件 |
| **URL** | 粘贴网页链接 |

### 5.3 生成播客

```bash
# 1. 选择文档
文档列表 → 选择要讨论的文档

# 2. 配置选项
- 播客时长：5/10/15/30 分钟
- 主持风格：学术/轻松/专业
- 讨论深度：入门/进阶/深入

# 3. 生成
点击 "Generate Podcast" → 等待生成 → 播放
```

### 5.4 交互问答

```bash
# 1. 打开问答模式
在播客播放界面点击 "Ask Questions"

# 2. 输入问题
"这篇论文的方法和之前的工作有什么不同？"

# 3. 获取回答
AI 会基于文档内容给出准确答案

# 4. 追问
可以继续追问获取更多信息
```

## 七，最佳实践

### 7.1 文档准备

```
✅ 格式规范
   - 使用标准 PDF 格式
   - 避免扫描件（用 OCR 处理）

✅ 内容结构
   - 有清晰的章节标题
   - 包含摘要和结论

✅ 长度适中
   - 单篇建议 10-50 页
   - 过长文档建议分段处理
```

### 7.2 播客配置

| 场景 | 配置建议 |
|------|----------|
| **学术论文** | 时长 10-15min，深度深入 |
| **产品文档** | 时长 5-10min，轻松风格 |
| **技术报告** | 时长 15-30min，专业风格 |

### 7.3 提问技巧

```
✅ 好的问题
   - "这篇论文的核心创新是什么？"
   - "方法和之前的工作有何不同？"
   - "实验结果说明了什么？"

❌ 避免
   - 过于宽泛的问题
   - 与文档无关的问题
   - 主观观点类问题
```
