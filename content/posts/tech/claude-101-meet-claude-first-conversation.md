## 学习目标

学完本节后，你将能够：

- 理解什么是"提示词"（Prompt）以及它为什么重要
- 掌握三种基本的提问方式：直接提问、角色扮演、格式约束
- 学会给 Claude 提供足够的背景信息
- 了解常见的提问错误以及如何避免

---

## 为什么提问方式很重要？

很多人第一次用 AI 时，会遇到这种情况：

> **用户**：帮我写一篇文章
> 
> **AI**（Claude）：好的，请问您想写什么主题的文章？长度是多少？读者是谁？

然后用户就懵了——他们期待的是一个完整的答案，而不是一连串问题。

这背后的原因是：**AI 没有读心术**。你不说清楚，它就只能猜测。而猜测往往不准确。

所以，**提问方式直接决定了回答质量**。学会正确提问，相当于掌握了打开 AI 宝库的钥匙。

---

## 三种基本提问方式

### 方式一：直接提问（最简单）

这是最基础的提问方式，适合简单的问题：

```
❌ 不好的例子
"Python"

✅ 好的例子
"Python 是什么？它跟 JavaScript 有什么区别？我是一个前端开发者，
想了解该不该学 Python。"
```

**技巧**：
- 说明你的身份（我是谁）
- 说明你要解决的问题
- 说明你已经知道了什么（减少它解释基础的时间）

---

### 方式二：角色扮演（很有效）

给 Claude 一个身份，能让它的回答更专业、更贴合需求：

```
❌ 普通提问
"怎么写一封辞职信？"

✅ 角色扮演
"你是一个有 10 年经验的高级 HR，看过上千封辞职信。
帮我写一封辞职信，要求：
- 语气得体，既表达感激又不显得卑微
- 长度 200 字以内
- 不用说明具体离职原因
- 格式规范，适合正式场合"
```

**什么时候用**：
- 需要专业意见时（律师、医生、会计等）
- 需要特定风格的回答时
- 希望获得更有深度的分析时

---

### 方式三：格式约束（最强大）

告诉 Claude 你希望用什么格式输出，能大幅提升实用性：

```
❌ 无格式要求
"给我一些健身建议"

✅ 有格式要求
"给我 5 个在家就能做的健身动作，每个动作包含：
- 动作名称
- 锻炼哪个部位
- 动作要领（3 条以内）
- 注意事项

用表格的形式输出"
```

**常用格式**：
- 表格（适合比较多个选项）
- 列表（适合步骤、要点）
- bullet points（适合快速浏览）
- 代码块（适合技术内容）
- 标题 + 正文（适合结构化内容）

## 提供背景信息的艺术

### 为什么背景信息很关键？

想象你问一个朋友：

> "今天吃什么？"

他可能会一脸茫然——他不知道你：
- 是饿了还是只是好奇
- 想省钱还是愿意花大钱
- 在家还是在外
- 有没有忌口

AI 也是一样的道理。

### 背景信息的黄金公式

```
我是谁 + 我的目标 + 我已经做了什么 + 约束条件
```

**举例**：

```
"我是一个大学生（下周要交期末论文），写了一篇关于人工智能伦理的
论文草稿（3000 字），但导师反馈说'论证不够深入'。我需要：
1. 理解为什么论证不深入
2. 3 个具体方法来加强论证
3. 每个方法给出具体例子

请用通俗易懂的方式解释，我不太懂哲学。"
```

## 常见错误和修正

### ❌ 错误一：问题太模糊

```
❌ "帮我写代码"
✅ "用 Python 写一个函数，接收一个列表，返回列表中所有偶数的和"
```

### ❌ 错误二：要求太模糊

```
❌ "帮我写一封邮件"
✅ "帮我写一封求职邮件，应聘产品经理岗位。强调我 3 年经验，
做过社交类产品，突出我的用户增长成绩。语气专业但不生硬。"
```

### ❌ 错误三：没有指定输出形式

```
❌ "介绍一下 React"
✅ "用 300 字介绍 React 的核心概念，适合有 HTML/CSS 基础的初学者。
最后列出 3 个适合新手的练手项目。"
```

### ❌ 错误四：问题太复杂

```
❌ "解释一下从计算机发明到互联网诞生的完整历史"
✅ 拆成多个问题：
1. "计算机是怎么发明的？"
2. "互联网是怎么诞生的？"
3. "Web 是怎么出现的？"
```

**提示**：把复杂问题拆成多个简单问题，AI 的回答质量会更高。

## 迭代式提问

有时候一次提问得不到理想答案，这是正常的。试试**迭代优化**：

### 第一轮：获取初步答案

```
用户：帮我制定一个 3 个月的学习计划，要学会 Python 爬虫
```

### 第二轮：补充信息

```
用户：第一周的内容太多了，我只能每天花 1 小时。还有，我没有编程基础，
从零开始。调整一下计划。
```

### 第三轮：进一步优化

```
用户：很好，但我想把数据分析也加进去。能不能把 3 个月改成 4 个月，
重新调整？
```

**关键心态**：把跟 AI 对话当成跟一个聪明的助手协作，不是一次性交易。

## 一个完整的提问模板

如果你不知道该怎么提问，可以试试这个模板：

```
【我的情况】
- 我是……（身份/背景）
- 我的问题是……（具体问题）
- 我已经尝试过……（已有的努力）

【我的目标】
- 我想要……（期望结果）

【约束条件】
- 篇幅/长度：……
- 格式：……
- 风格：……
- 其他要求：……

【我不想要的】
- ……（明确排除的内容或风格）
```

**示例**：

```
【我的情况】
- 我是一个想转行做程序员的产品经理，完全没有编程基础
- 我的问题是如何从零开始学习 Python

【我的目标】
- 我想要一个详细但实际可行的学习路线

【约束条件】
- 每天学习时间 2 小时以内
- 3 个月后能找到 Python 相关工作
- 偏向 web 开发方向

【我不想要的】
- 不要太多理论，要多实践
- 不要花哨的技术名词，用通俗语言
```

## 练习

### 练习一：改造你的问题

把下面这个模糊的问题，改成具体清晰的问题：

> "帮我看看这段代码有什么问题"  
> （没有贴代码，也没有说预期行为）

### 练习二：角色扮演

用角色扮演的方式，向 Claude 咨询：你想跳槽，但它建议你先别跳——你该如何追问让它给你更有针对性的建议？

### 练习三：格式约束

让 Claude 用「思维导图的文字形式」解释什么是设计模式（即用缩进和符号模拟树状结构）。
