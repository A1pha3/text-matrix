## 学习目标

学完本节后，你将能够：

- 知道如何自然地开始与 Claude 的第一段对话，而不是被“提示词工程”吓住。
- 理解一条好消息最少要包含什么信息，避免让 Claude 只能猜你的需求。
- 学会通过追问、补充背景、要求重写，让对话一轮轮变得更好。
- 能把 Claude 用在总结、写作、解释、头脑风暴、计划制定等日常工作任务里。

---

## 什么时候这节课最有价值

这节课最适合下面三类人：

### 场景一：你知道 Claude 很强，但一打开输入框就不知道第一句怎么说

这类卡点很常见。问题通常不在于你不会“高级提示词”，而在于你还没有形成最基本的对话起手式。

### 场景二：你已经问过几次，但总觉得回答“有点用，又不太够用”

这往往说明你缺的不是模型能力，而是把任务、背景和结果形式说清楚的习惯。

### 场景三：你想把 Claude 真正带进日常工作，而不是偶尔玩一下

第一次对话不是一次演示，而是未来所有协作方式的起点。开场方式对了，后续的迭代成本会低很多。

---

## 先别追求“完美提示词”

很多人第一次打开 Claude，脑子里都会冒出同一个问题：

> “我是不是得先学一整套高级提示词技巧，才能开始用？”

答案是否定的。

这节课更接近原课程标题强调的重点：**先开始对话**。你不需要一上来就掌握复杂的角色设定、Few-shot、结构化标签这些进阶技巧。那部分内容更适合放到下一课 **Getting Better Results** 中系统学习。

第一段对话真正重要的，是建立一个正确心智模型：

- **把 Claude 当成可协作的助手，不是搜索框。**
- **先用自然语言说明你的目标，再逐步补充细节。**
- **第一轮不完美很正常，关键是会继续往下聊。**

这也是为什么课程名字不是“Write the perfect prompt”，而是 **Your first conversation with Claude**。

## Claude 适合从哪些对话开始？

如果你完全不知道第一句话该发什么，可以从最容易得到价值的 5 类任务开始：

| 场景 | 你可以直接这样开口 | 为什么适合作为第一段对话 |
| :--- | :--- | :--- |
| **总结信息** | “帮我总结这段会议纪要，提炼 3 个结论和 3 个待办事项。” | 输入明确，反馈快，容易感受到 Claude 的价值 |
| **改写文字** | “把这封邮件改得更礼貌，但不要太生硬。” | 结果直观，很容易判断好不好 |
| **解释概念** | “请用产品经理能听懂的话解释什么是 API。” | 能快速体验 Claude 的讲解能力 |
| **头脑风暴** | “我想给一个记账 App 起名字，先给我 20 个方向。” | 适合低风险探索，不怕多试几轮 |
| **制定计划** | “我想在 4 周内重新学习 SQL，帮我做一个可执行计划。” | 能感受到 Claude 在组织思路上的优势 |

你会发现，这些开场白都不复杂。它们的共同点不是“高级”，而是**有明确目标**。

## 第一条消息，最少说清 3 件事

第一次和 Claude 对话，不必写成长篇大论，但最好至少说清下面 3 件事：

1. **你想完成什么**
2. **Claude 需要知道哪些背景**
3. **你希望得到什么形式的结果**

你可以把它理解成一个很轻量的入门公式：

`目标 + 背景 + 结果形式`

### 对比看看

```text
❌ 太模糊
“帮我写一下。”

✅ 更容易得到好结果
“我要给客户发一封项目延期说明邮件。
背景是：延期 3 天，原因是测试阶段发现了一个高优先级 bug，已经在修。
请帮我写成专业、诚恳但不过度道歉的语气，控制在 200 字以内。”
```

### 为什么这样有效？

因为 Claude 没有读心术。你不说目标，它就不知道你要写邮件、做摘要还是做方案；你不说背景，它就只能靠猜；你不说结果形式，它就可能给你一大段不方便直接使用的回答。

换句话说，**一条好消息不是“更高级”，而是“更少让模型猜”**。

---

## 第一段对话的正确节奏：说一句，收一轮，再推进

很多新手有一个误区：想在第一条消息里一次性把所有要求都塞完。

其实更自然的做法是：

### 第一步：先拿到一个可用版本

```text
“帮我做一个 1 周的日本旅行行程，重点放在东京和镰仓。”
```

### 第二步：指出哪里不合适

```text
“这个行程有点太赶了。我不想每天换酒店，而且我更喜欢逛书店和咖啡馆，不太想去太多热门打卡点。”
```

### 第三步：要求按新条件重做

```text
“请保留东京和镰仓，但改成节奏慢一点的版本，并按每天上午、下午、晚上分段输出。”
```

这种方式比一开始就写 300 字提示词更接近日常沟通，也更符合“conversation”这个词本身的含义。

## 真实示例：把“问一句”聊成“拿结果”

下面是一个更完整的例子，展示第一段对话是怎么逐步变好的。

```text
第一轮
用户：
“我要做一个关于 AI 办公工具的分享，帮我列一个大纲。”

Claude：
“可以，下面是一个关于 AI 办公工具的分享大纲……”
```

这时你大概率会发现：它有帮助，但还不够贴合。

```text
第二轮
用户：
“这次分享对象是公司里的运营和销售同事，不是技术团队。
请把术语减少一些，重点放在怎么提升日常效率，而不是模型原理。”
```

Claude 的回答通常会立刻变得更贴近你的真实场景。

```text
第三轮
用户：
“很好。现在请把大纲改成 15 分钟分享版本，并补一个结尾页：
1. 今天就能开始做的 3 件小事
2. 使用 AI 时最常见的 3 个风险”
```

到这里，你已经完成了一次非常典型的 Claude 协作流程：

- 用第一轮拿到草稿
- 用第二轮校正方向
- 用第三轮打磨成可直接交付的版本

这比“第一次就问到完美”更重要，也更符合真实工作流。

## 你可以怎样继续追问？

当第一轮答案不够理想时，不要急着重开对话。优先试试下面 4 种追问方式：

```text
1. 补充背景
“补充一点背景：我们的预算只有 5 万元，且团队只有 2 个人。”

2. 调整目标
“不要再追求全面，请优先给我一个今天就能执行的版本。”

3. 指出问题
“这版太空泛了，缺少具体步骤。请改成可以直接照着做的清单。”

4. 改变表达形式
“保留内容不变，改成表格输出。”
```

这 4 种追问方式，本质上都在做一件事：**减少歧义，缩小任务范围，让 Claude 更接近你的真实需求**。

## 日常工作里，第一段对话可以怎么用？

Claude 101 课程简介里提到，它特别适合 everyday work tasks。落到具体操作上，你可以这样开始：

### 1. 写作起草

```text
“我需要写一段产品更新公告。语气积极一点，但不要像广告文案。”
```

### 2. 总结冗长内容

```text
“下面是一次 40 分钟会议的整理稿。请先总结结论，再列待办事项。”
```

### 3. 学习新概念

```text
“请把向量数据库解释给一个会写 SQL、但没接触过 AI 的人听。”
```

### 4. 做头脑风暴

```text
“我要做一个面向自由职业者的记账产品，先给我 10 个差异化切入点。”
```

### 5. 辅助规划任务

```text
“我下周要完成招聘面试、季度复盘、客户回访三件事，请帮我排一个优先级计划。”
```

如果你不知道 Claude 对什么最有帮助，就从这些低门槛、高反馈的任务开始。

## 常见误区：第一次用 Claude 时，最容易踩的 5 个坑

| 误区 | 表现 | 更好的做法 |
| :--- | :--- | :--- |
| **把 Claude 当搜索框** | 只扔一个关键词，希望它自己猜 | 用完整句子说清任务和目的 |
| **第一轮就追求完美** | 一次塞进过多要求，结果自己也看不懂 | 先拿草稿，再继续修 |
| **不给背景** | 忽略对象、场景、限制条件 | 至少补充身份、受众、限制 |
| **答案不满意就重开** | 每次都从头来过 | 直接在原对话里指出问题、继续追问 |
| **把 AI 当最终裁判** | 对输出照单全收 | 重要事实、数据、结论要自己复核 |

## 一个入门即用的小模板

如果你面对空白输入框时依然有点紧张，可以先用这个非常轻量的模板：

```text
我想让你帮我完成：[任务]

背景是：[必要上下文]

请给我：[你想要的结果形式]
```

例如：

```text
我想让你帮我完成：准备一次部门周会发言

背景是：我要汇报本周项目进展、风险和下周计划，听众是产品经理和销售负责人

请给我：一个 3 分钟可直接照着讲的发言提纲
```

先把这个模板用顺手，比背一堆高级术语更重要。

## 行动建议：把第一段对话变成你的固定开场

很多人读完这节后，最大的误区是“我懂了”，但下次打开 Claude 还是不知道怎么开始。

更有效的做法是把本节内容沉淀成你自己的 3 个固定开场模板，例如：

1. **总结类**：目标 + 材料 + 输出结构
2. **写作类**：任务 + 受众 + 语气 + 长度
3. **规划类**：目标 + 资源限制 + 时间范围 + 交付形式

这样你以后不是“重新想一句话”，而是直接从一个稳定开场开始，再继续追问。

## 常见问题

### Claude 回答偏了，怎么办？

不要只说“再来一版”。更有效的方式是指出偏差：

```text
“方向不对。我不是要战略分析，而是要今天能执行的行动清单。”
```

### 一开始就要很会写提示词吗？

不需要。先会说清任务，再逐步学会补背景、控格式、做迭代，就已经能超过大多数初学者。

### 可以直接把长文档贴给 Claude 吗？

可以，但最好先告诉它你要什么结果，例如“先总结结论，再列风险，再给建议”。否则它可能只会给你一个过于泛化的摘要。

## 动手练习

请真的打开 Claude，完成下面 3 个小练习。重点不是“一次成功”，而是体验“对话会逐轮变好”。

1. **从自然语言开始**
   打开 Claude，直接发送一句你今天真实需要帮助的话，例如：
   “帮我整理今天的待办事项。”
   然后观察：它最先帮你解决的是什么？

2. **做一次追问**
   在上一轮回答后，再补一句限制条件，例如：
   “我今天只有 2 小时，而且最重要的是客户相关任务。”
   对比两轮输出有什么变化。

3. **把结果变成可执行**
   再继续追问：
   “很好。请改成按优先级排序的清单，每项后面标上预计花费时间。”
   体验 Claude 如何从“聊天”变成“协作”。
